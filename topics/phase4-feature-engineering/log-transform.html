Log Transform

Income, house prices, and city populations all follow the same pattern — a huge pile of small values and a long tail of rare large values. The log transform compresses that long tail, making these right-skewed distributions bell-shaped and easier for models to learn from.

16 min Beginner numpy · pandas

ln
The idea — squishing the long tail

Imagine house prices in a city: most sell for $200k–$600k, but a handful of mansions sell for $5M–$20M. The model keeps seeing huge salary numbers and struggles. The log transform squishes big numbers down while barely touching small ones — creating a more balanced distribution.

Log compresses large numbers more than small ones
Key insight: Going from $1 to $10 (10× bigger) adds the same log amount as going from $10,000 to $100,000 (also 10× bigger). Log measures multiplication factors, not absolute differences.

When to use it — and when NOT to

✓ Use log transform when
✓ Skewness > 1 or < -1 (highly skewed)
✓ Income, salary, house prices
✓ Population, page views, counts
✓ Values span multiple orders of magnitude
✓ Linear regression or models that assume normality
✓ Feature or target variable is right-skewed
✗ Do NOT use log when
✗ Data has zeros — log(0) = −∞ (use log1p instead)
✗ Data has negative values — log of negative is undefined
✗ Data is already normally distributed
✗ Tree-based models (RF, XGBoost) — don't need it
✗ You need to interpret coefficients in original scale
✗ Data is left-skewed (try square or cube instead)

Handling zeros and negatives

log1p — for zeros
log1p(x) = log(1 + x)
Adds 1 before taking log. log1p(0) = log(1) = 0. Safe for count data like "number of purchases" which can be 0.
shift + log — negatives
log(x − min + 1)
Shift all values up so minimum becomes 1, then log. Works when data has negatives (like temperature in Celsius).
signed log — both sides
sign(x) · log(|x| + 1)
For data that can be positive or negative (profits/losses). Preserves sign while compressing magnitude.

How to code it

1
Check skewness first — only transform if needed
Don't transform blindly. Check the skewness value. If |skew| > 1, consider log transform.
from scipy.stats import skew print(df['income'].skew()) # > 1 → right skewed → try log import matplotlib.pyplot as plt df['income'].hist() # visual check
2
Apply the right variant
import numpy as np # All values > 0: use log df['log_income'] = np.log(df['income']) # Has zeros: use log1p df['log_views'] = np.log1p(df['page_views']) # sklearn: FunctionTransformer from sklearn.preprocessing import FunctionTransformer log_transformer = FunctionTransformer(np.log1p, validate=True)
3
In a sklearn Pipeline
from sklearn.pipeline import Pipeline from sklearn.preprocessing import FunctionTransformer, StandardScaler pipe = Pipeline([ ('log', FunctionTransformer(np.log1p)), # log1p first ('scaler', StandardScaler()), # then standardize ('model', LinearRegression()) ])
4
Reverse the transform for predictions
If you log-transformed the target variable (y), remember to exponentiate predictions to get back original scale.
y_pred_log = model.predict(X_test) y_pred_original = np.expm1(y_pred_log) # reverse of log1p # np.exp() reverses np.log() # np.expm1() reverses np.log1p()

Common mistakes

x
Applying log to data with zeros or negatives. log(0) = −∞ and log(negative) = error. Always check: df['col'].min(). If min is 0, use log1p. If min is negative, shift first.
x
Forgetting to reverse the transform on predictions. If you log-transformed the target y, model predictions are in log space. Use np.expm1() or np.exp() to get predictions back in original units.
x
Applying log to already-normal data. Log transform on symmetric data makes it left-skewed. Always check skewness first — only apply if |skew| > 1.
x
Using log on tree-based models unnecessarily. Random Forest, XGBoost, LightGBM are scale and distribution invariant. Log transforming features doesn't help them and makes features harder to interpret.

*
Key takeaways

Remember these 5 things
Purpose: converts right-skewed distributions (income, prices, counts) into approximately normal ones
Has zeros? Use np.log1p(x) not np.log(x). log1p(0) = 0, log(0) = error
Check skewness first: only apply when |skew| > 1. Don't transform symmetric data
Reverse it: predictions on log-transformed y need np.expm1() to return to original scale
Not needed for: tree-based models (RF, XGBoost) — they don't care about distribution shape
