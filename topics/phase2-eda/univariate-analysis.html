IQR method
outlier if x < Q1-1.5*IQR or x > Q3+1.5*IQR
Robust to outliers. Best for skewed distributions. Industry standard for box plots.
Z-score method
z = (x - mean) / std outlier if |z| > 2.5
Assumes normal distribution. Sensitive to outliers (mean/std are affected). Use for roughly normal data.
Modified Z-score
mz = 0.6745*(x-median)/MAD outlier if |mz| > 3.5
Uses median and MAD. Most robust method. Works even when many outliers present. Best for contaminated data.

Common mistakes

x
Using mean for skewed distributions. For income, house prices, or any right-skewed data, the mean is pulled by outliers. Report median + IQR instead of mean + std.
x
Ignoring the bin width effect on histograms. Changing bin width can make the same data look normal, bimodal, or uniform. Always try multiple bin widths and use KDE alongside.
x
Removing all outliers automatically. Outliers are data too. First investigate why they exist. A salary of $10M may be real data for a CEO. Only remove if there is a clear reason (data entry error, measurement fault).
x
Forgetting to check for rare categories. A category with 3 occurrences in 10,000 rows will cause problems in train/test splits and cross-validation. Group rare categories into "Other".
x
Using a bar chart for continuous data. Continuous variables need histograms or KDE plots. Bar charts are for categorical counts. Using a bar chart for age is technically wrong.

*
Key takeaways

Summary
Always start with df.describe() and histogram for numerical, value_counts() for categorical.
For skewed data: use median and IQR, not mean and std. Log-transform before modelling.
Use IQR method for outlier detection. Investigate before removing — outliers may be real.
Check skewness and kurtosis numerically: |skew| > 1 means significantly skewed.
For categorical: check cardinality, rare categories (<5%), and dominant categories (>80%).
Look at distribution shape: normal, right-skewed, left-skewed, bimodal, or uniform each imply different actions.
