Common mistakes

x
Including highly correlated features in linear models. When |r| > 0.85 between two features, the coefficients become unstable. The model cannot distinguish their individual effects. Use VIF to detect this.
x
Not scaling before PCA. PCA finds directions of maximum variance. If income (0-100000) and age (0-100) are both in the data, PCA will be dominated by income. Always StandardScale before PCA.
x
Using correlation for feature selection with tree models. Correlation only captures linear relationships. Tree-based models (RF, XGBoost) can use non-linear features. Use mutual information or permutation importance instead.
x
Dropping features based on low correlation with target alone. A feature with r=0.05 with the target might still be important in combination with other features (interaction effects). Use model-based feature importance, not just correlation.
x
Choosing n_components for PCA without a scree plot. Always plot the cumulative explained variance. Pick the number of components where the curve bends (the elbow) or where cumulative variance exceeds 90-95%.

*
Key takeaways

Summary
Correlation heatmap: spot multicollinearity (|r| > 0.85) and which features correlate with the target.
Parallel coordinates: visualise all features at once, coloured by class — reveals separability and patterns.
Always StandardScale before PCA. Choose n_components via the scree plot elbow or 90% cumulative variance.
VIF > 10 = severe multicollinearity. Drop one of the correlated features or use Ridge/PCA regression.
Use multiple feature importance methods — correlation, mutual information, and model-based importance can give very different rankings.
Multivariate EDA is about finding structure before modelling — do not skip it even under time pressure.
