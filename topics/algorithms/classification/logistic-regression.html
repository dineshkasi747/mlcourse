Logistic Regression

Despite the name, it's a classifier. It squashes any real number into a probability between 0 and 1 using the sigmoid function, then draws a decision boundary where that probability crosses 0.5.

Key insight: The coefficients of logistic regression are slopes on the log-odds scale. A coefficient of 0.5 means each unit increase in the feature multiplies the odds by e⁰·⁵ ≈ 1.65. This is why we exponentiate coefficients to get odds ratios.

Why not MSE? Log-loss explained visually

Logistic regression uses binary cross-entropy (log-loss), not MSE. Here's why: MSE applied to a sigmoid creates a non-convex loss surface with many local minima — gradient descent gets stuck. Log-loss is always convex.

✕ MSE loss — NON-CONVEX (bad)
✓ Log-loss — CONVEX (good)
Log-loss: cost of predicting wrong probability

Move along the curve to see how the loss explodes as you predict confidently wrong. Predicting 0.01 when y=1 incurs a loss of ~4.6 — nearly 5× worse than uncertain prediction.

∑
Full derivation — 5 steps

1
Linear combination → z (the logit)
Start with a standard linear combination of inputs. This can be any real number.
z = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ = wᵀx + b
2
Apply sigmoid → P(y=1|x)
Squash z into (0,1) using the sigmoid. This is the predicted probability.
ŷ = σ(z) = 1 / (1 + e⁻ᶻ)
3
Bernoulli likelihood
Model each training example as a Bernoulli trial. P(yᵢ|xᵢ) = ŷᵢ if yᵢ=1, else (1−ŷᵢ). Combined compactly:
P(y|x) = ŷʸ · (1 − ŷ)⁽¹⁻ʸ⁾
4
Negative log-likelihood = binary cross-entropy
Maximise likelihood ≡ minimise negative log-likelihood. Taking log turns products into sums (numerically stable). Dividing by N gives mean loss.
J(w) = −(1/N) Σ [ yᵢ log(ŷᵢ) + (1−yᵢ) log(1−ŷᵢ) ]
5
Gradient descent update rule
The gradient ∂J/∂w has a beautiful clean form — almost identical to linear regression. Update repeatedly until convergence.
∂J/∂w = (1/N) · Xᵀ(ŷ − y) w ← w − α · ∂J/∂w b ← b − α · (1/N) Σ (ŷᵢ − yᵢ)
✨

Beautiful symmetry: The gradient update rule (1/N) · Xᵀ(ŷ−y) is identical to linear regression's gradient — just replace the linear prediction with sigmoid(Xw+b). This is why logistic regression is intuitive if you already know linear regression.

Multiclass — One-vs-Rest and Softmax

Logistic regression handles 3+ classes via two strategies. Understanding the difference matters for choosing the right one.

One-vs-Rest (OvR)
Default in sklearn
Train K separate binary classifiers — one per class, treating that class as positive and all others as negative. At prediction time, pick the class whose classifier fires with highest confidence.
K classifiers × N iterations Probabilities don't sum to 1
Good for: Large K, when classes are well-separated.
Softmax Regression
multi_class='multinomial'
One unified model. Outputs a probability distribution over all K classes that sums to exactly 1. Uses cross-entropy loss over all classes simultaneously.
softmax(z)ₖ = e^zₖ / Σ e^zⱼ Probabilities sum to exactly 1
Good for: Mutually exclusive classes, when K is small.

Common mistakes

✕
Using logistic regression on non-linear problems without feature engineering. LogReg draws a straight decision boundary. For circular or XOR patterns, add polynomial features or switch to a non-linear model.
✕
Interpreting coefficients as probabilities. Coefficients are log-odds ratios, not probability changes. A coefficient of 2 means multiplying odds by e² ≈ 7.4 — not a 200% probability increase.
✕
Not scaling features. LogReg converges much faster when features are on the same scale. StandardScaler is strongly recommended, especially when comparing coefficient magnitudes.
✕
Using default threshold=0.5 on imbalanced data. With 5% positives, everything gets labelled negative. Lower the threshold or use class_weight='balanced'. Always check precision/recall, not just accuracy.
✕
Ignoring perfect separation / complete separation. If a single feature perfectly separates classes, the coefficient grows to infinity and sklearn will warn. Use regularisation (C parameter) or add more data.

★
Key takeaways

Summary
Logistic regression uses the sigmoid to squash a linear combination into (0,1), producing a probability. It draws a linear decision boundary.
It optimises binary cross-entropy (log-loss) — not MSE. Log-loss is convex, guaranteeing global minimum with gradient descent.
Coefficients are log-odds ratios. Exponentiate them (e^w) to get odds ratios — how much the odds multiply per unit increase in a feature.
Threshold is a business decision. 0.5 is not always right — choose based on the cost of false positives vs false negatives in your application.
Always check your ROC-AUC score, not just accuracy. AUC = probability that the model ranks a random positive higher than a random negative.
Scale features before training. Use C parameter (inverse of regularisation strength) to prevent overfitting. Smaller C = stronger regularisation.
