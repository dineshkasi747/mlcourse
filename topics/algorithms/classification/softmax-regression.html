Softmax Regression

The natural extension of logistic regression to K classes. Instead of one sigmoid, run K linear functions and normalise them into a probability distribution that always sums to 1.

How softmax works — step by step

An interactive walkthrough of the forward pass for a single input sample with 3 classes.

Forward pass trace — drag sliders to change input
1
Linear scores
Compute one raw score per class
For each class k, compute a linear score (logit) z_k = w_k . x + b_k. Each class has its own weight vector w_k. These raw scores can be any real number — positive or negative.
z_k = w_k . x + b_k for k = 1..K Z = W @ x + b # matrix form: shape (K,)
2
Exponentiate
Take e^z for each logit
Exponentiation makes all values positive and amplifies differences between scores. A logit 2 units higher becomes e^2 = 7.4x more probable. This is the key step that gives softmax its "winner-amplifying" property.
exp_z_k = exp(z_k) for k = 1..K # Numerically stable version (subtract max first): exp_z_k = exp(z_k - max(z))
3
Normalise
Divide by the sum to get probabilities
Divide each exponentiated value by the total sum. This ensures all probabilities are in (0,1) and sum to exactly 1.0 — a valid probability distribution.
p_k = exp(z_k) / sum_j exp(z_j) softmax(z)_k = exp(z_k) / sum_j exp(z_j) # All p_k in (0, 1) and sum(p) = 1.0 always
4
Cross-entropy loss
Penalise low probability on the true class
The loss is -log(p_true). If the model assigns p=0.9 to the correct class: loss = -log(0.9) = 0.105. If it assigns p=0.1: loss = -log(0.1) = 2.30. Low confidence on the right answer is heavily penalised.
L = -log( p_true ) = -log( softmax(z)[y] ) # For a batch of n samples: L = -(1/n) * sum_i log( p[i, y_i] )

Softmax vs Logistic Regression

Logistic Regression
Binary only — 2 classes
p(y=1) = sigmoid(w.x + b) sigmoid(z) = 1/(1+e^-z) Output: single prob in (0,1)
One weight vector w. Output is P(y=1) directly. P(y=0) = 1 - P(y=1) is implied.
Softmax Regression
Multiclass — K classes
p(y=k) = softmax(W.x+b)[k] softmax(z)_k = e^z_k/sum(e^z) Output: K probs summing to 1
K weight vectors (one per class). Output is a full probability distribution over all K classes simultaneously.
i

Logistic regression is a special case of softmax with K=2. When K=2, softmax reduces to exactly sigmoid on the score difference (z_1 - z_0). In sklearn, LogisticRegression with multi_class='multinomial' is softmax regression.

Common mistakes

x
Numerical overflow: exp(1000) = inf. Never compute softmax as exp(z)/sum(exp(z)) directly on raw logits. Always subtract max(z) first: softmax(z) = softmax(z - max(z)). Mathematically identical, numerically safe.
x
Using softmax + BCE loss (double softmax). PyTorch's CrossEntropyLoss applies log_softmax internally. If you add a softmax layer before it, you softmax twice and the gradients are wrong. Use raw logits with CrossEntropyLoss.
x
Treating softmax probabilities as calibrated confidence. A model outputting p=0.95 does not mean 95% accuracy. Softmax probabilities are often overconfident. Use temperature scaling or Platt scaling to calibrate.
x
Using softmax for multi-label problems. Softmax assumes exactly one class is correct (mutually exclusive). For multi-label (a photo can be both "cat" AND "outdoor"), use sigmoid on each output independently.

Key takeaways

Summary
Softmax converts K raw scores (logits) into a probability distribution that always sums to 1.
Formula: p_k = exp(z_k) / sum_j exp(z_j). Always subtract max(z) first for numerical stability.
Trained with cross-entropy loss: L = -log(p_true). Low confidence on the correct class is heavily penalised.
Temperature T controls sharpness: T<1 sharpens (overconfident), T=1 standard, T>1 flattens (uncertain).
Logistic regression is softmax with K=2. In sklearn, use multi_class='multinomial' for softmax.
Use sigmoid (not softmax) for multi-label problems where multiple classes can be true simultaneously.
