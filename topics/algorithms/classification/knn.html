Algorithm — step by step

1
Store all training data
KNN has no training phase. It simply memorises every (X, y) pair. This is called a lazy learner — all work happens at prediction time.
model.fit(X_train, y_train) # just stores X, y
2
Receive a new query point
At prediction time, receive unseen point x_query. We need to find what class it belongs to.
x_query = [5.1, 3.5, 1.4, 0.2] # new flower measurements
3
Compute distance to every training point
Calculate the distance from x_query to all N training points using the chosen metric. This is the expensive step — O(N) per query.
d(x, y) = sqrt( sum( (xi - yi)^2 ) ) # Euclidean
4
Sort and select K nearest
Sort all distances ascending. Take the K training points with the smallest distances — these are the K nearest neighbours.
neighbours = argsort(distances)[:K]
5
Majority vote (classification) or mean (regression)
For classification: the most common class among K neighbours wins. For regression: return the average of K neighbours' values.
prediction = mode(y[neighbours]) # classification prediction = mean(y[neighbours]) # regression

Common mistakes

✕
Not scaling features. If age (0-100) and salary (0-100000) are both raw, salary dominates all distances. Always StandardScale or MinMaxScale first.
✕
K=1 in production. Memorises training data perfectly, generalises terribly. One noisy point becomes a decision region. Use CV to tune K.
✕
Using KNN on high-dimensional data. The curse of dimensionality makes all points equidistant above ~20 features. Use PCA first.
✕
Forgetting prediction cost. KNN is O(N) per query — with 10M training points and 100K queries, that is 1 trillion distance calculations. Use KD-tree or Ball-tree (default in sklearn).
✕
Even K for binary classification. K=4 with 2 classes tied 2-2 causes random tie-breaking. Always use odd K for 2 classes.

★
Key takeaways

Summary
KNN stores all training data and classifies by majority vote of K nearest neighbours at prediction time.
Always scale features before KNN — distance is meaningless when features have different units.
Small K = complex boundary (overfits). Large K = smooth boundary (underfits). Tune with cross-validation.
KNN has no training cost but O(N) prediction cost. Use KD-tree or Ball-tree for large N.
Works for both classification (vote) and regression (average).
Struggles above ~20 features due to the curse of dimensionality — apply PCA first.
