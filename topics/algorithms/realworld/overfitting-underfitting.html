Overfitting & Underfitting

The two fundamental failure modes of machine learning. Underfitting means your model is too simple to learn the pattern. Overfitting means it memorised the training data so well that it fails on new data. The goal is the sweet spot in between — good generalisation.

Bias-variance tradeoff — the theoretical view

Every model's error can be decomposed into three parts. Understanding this decomposition explains why overfitting and underfitting happen and how to fix them.

Bias — systematic error
Bias = E[ŷ] - y
How wrong the model is on average across many different training sets. High bias = model is too simple, can't capture the true pattern. Causes underfitting. Linear model fitting a sine curve has high bias.
Variance — sensitivity to data
Variance = E[(ŷ - E[ŷ])²]
How much predictions change when trained on different subsets of data. High variance = model fits noise in the training set. Causes overfitting. Degree-15 polynomial changes wildly with each new dataset.
Irreducible noise
Noise = inherent randomness in y
Error that no model can eliminate — it's the natural randomness in the data. A hospital dataset has measurement noise. Stock prices have genuine unpredictability. Cannot be reduced, only accepted.
Total error
Error = Bias² + Variance + Noise
Reducing bias usually increases variance and vice versa. The art of ML is finding the model complexity where the sum is minimised. Cross-validation finds this balance empirically.

How to fix overfitting and underfitting

OF
Fix Overfitting
  • Get more training data (most effective)
  • Add regularisation (L1/L2/Dropout)
  • Reduce model complexity (fewer layers, lower degree)
  • Early stopping (stop before train loss diverges from val)
  • Data augmentation (artificially expand dataset)
  • Cross-validation to choose correct hyperparameters
  • Feature selection (remove noisy/irrelevant features)
  • Ensemble methods (average predictions — reduces variance)
UF
Fix Underfitting
  • Increase model complexity (more layers, higher degree)
  • Train for longer (more epochs / iterations)
  • Add more informative features (feature engineering)
  • Reduce regularisation strength (smaller L1/L2)
  • Try a more powerful model class
  • Lower learning rate for more careful training
  • Remove incorrect assumptions (wrong loss function)

Data size — more data always helps overfitting

One of the most reliable cures for overfitting is simply more data. A complex model overfits when the training set is small because there is enough model capacity to memorise every point. More data forces the model to generalise.

Key takeaways

Summary
Underfitting = high bias. Both train and test error are high. Model is too simple for the problem.
Overfitting = high variance. Train error is low but test error is high. Model memorised noise.
The train-test gap is the clearest overfitting signal. Monitor it on every model you build.
More data is the most effective overfitting fix. Regularisation (L1/L2), early stopping, and dropout are the next best tools.
Total error = Bias² + Variance + Irreducible noise. You can't beat irreducible noise — focus on balancing bias and variance.
Use cross-validation to find the sweet spot in model complexity without touching the test set.
