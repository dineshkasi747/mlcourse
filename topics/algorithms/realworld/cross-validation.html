Cross Validation

A single train-test split gives a noisy estimate of model performance. CV uses data more efficiently by training and evaluating K times on different splits — giving a reliable estimate with confidence intervals.

Why K=5 or K=10? K=5 trains on 80% per fold (fast, efficient). K=10 trains on 90% (more accurate estimate, slower). LOOCV (K=N) is unbiased but extremely slow. K=5 is the standard default.

CV strategies — choose the right one

K-Fold
Default choice
Equal-sized rotating folds. Best for regression and balanced classification.
Stratified K-Fold
Imbalanced classes
Preserves class proportions in every fold. Always use for classification.
LOOCV
Small datasets
Each sample is validated once. K = N. Use when N < 100 samples.
Time Series CV
Sequential data
Validation always in future of training. Never shuffle time series data.
Group K-Fold
Grouped data
Keeps all samples from same patient/user in the same fold. Prevents group leakage.
Repeated K-Fold
High variance
K-Fold repeated with different random seeds. Reduces variance of the estimate.

How CV works — step by step

1
Split
Shuffle and divide into K equal folds
Shuffle the data (unless time series) then split into K approximately equal folds. With StratifiedKFold, class ratios are preserved in every fold automatically.
kf = KFold(n_splits=5, shuffle=True, random_state=42) for train_idx, val_idx in kf.split(X): X_train, X_val = X[train_idx], X[val_idx]
2
Train and evaluate
Fit on K-1 folds, score on held-out fold
Train the model from scratch on the training folds. Evaluate on the validation fold only. Record the score. Repeat for all K folds. The model never sees validation data during training.
scores = [] for train_idx, val_idx in kf.split(X): model.fit(X[train_idx], y[train_idx]) scores.append(model.score(X[val_idx], y[val_idx]))
3
Aggregate
Average scores and compute standard deviation
Mean score = CV performance estimate. Standard deviation = stability indicator. High std means the model is sensitive to which data it trains on.
cv_mean = np.mean(scores) # e.g. 0.856 cv_std = np.std(scores) # e.g. 0.023 # Report: 0.856 +/- 0.023
4
Final model
Retrain on ALL data after CV selection
CV gives a performance estimate. After picking the best model/hyperparameters via CV, retrain on the full dataset before deployment.
final_model = best_model.fit(X_all, y_all) # Deploy this - trained on all available data
5
Critical: avoid leakage
Preprocessing must be inside the CV loop
Scaling or encoding BEFORE CV leaks validation information into training. Always use Pipeline so preprocessing is fitted only on training folds each iteration.
pipe = Pipeline([ ('scaler', StandardScaler()), # fitted per fold only ('model', RandomForestClassifier()) ]) scores = cross_val_score(pipe, X, y, cv=5) # correct!

Common mistakes

x
Data leakage: preprocessing before CV. StandardScaling the full dataset before CV means the scaler has seen validation data. Always put preprocessing inside a Pipeline.
x
Not using StratifiedKFold for classification. Plain KFold on imbalanced data can create folds with no minority samples. Always use StratifiedKFold for classification.
x
Shuffling time series data. Never use shuffle=True on time series. Past must be training, future must be validation. Use TimeSeriesSplit.
x
Using CV score as the final test score. If CV was used to pick hyperparameters, run a final held-out test set to get an unbiased performance number.
x
Not retraining on all data after CV. The CV models each used K-1/K of the data. After CV selects the best model, retrain on ALL data before deployment.

*
Key takeaways

Summary
CV gives a more reliable performance estimate than a single split by averaging over K different data partitions.
Always use StratifiedKFold for classification. Never shuffle time series data - use TimeSeriesSplit.
Put ALL preprocessing inside a Pipeline to prevent data leakage from validation to training folds.
K=5 is the default (80% train per fold). K=10 for more accuracy. LOOCV for tiny datasets (<100 samples).
After CV picks the best model, retrain on ALL data before deployment.
Use nested CV for unbiased hyperparameter search: outer CV for performance estimate, inner CV for tuning.
