XGBoost

eXtreme Gradient Boosting — gradient boosting done right. XGBoost adds second-order optimisation, L1/L2 regularisation, column subsampling, and histogram-based splits that make it 10–100x faster than sklearn GBM with better accuracy. The algorithm that dominated Kaggle for a decade.

What makes XGBoost different from vanilla GBM?

Sklearn’s GradientBoostingClassifier is single-threaded and uses only first-order gradients. XGBoost adds four key innovations that make it dominate on tabular data:

1. Second-order gradients
Uses both gradient (g) and Hessian (h) at each step — like Newton’s method vs vanilla gradient descent. Converges faster.
2. L1 + L2 regularisation
reg_lambda (L2) and reg_alpha (L1) directly penalise leaf weights. Reduces overfitting without needing max_depth limits.
3. Column subsampling
colsample_bytree randomly samples features per tree — like Random Forest on top of boosting. Reduces correlation between trees.
4. Histogram splits
tree_method=hist bins continuous values into buckets before finding splits — reduces O(n×d) to O(bins×d) per tree. 10–100x faster.

The key maths — why second-order is better

Vanilla GBM fits a tree to the negative gradient of the loss. XGBoost uses a second-order Taylor expansion so each tree finds the optimal leaf weights analytically.

1
Taylor expansion of the loss
Approximate the loss after adding a new tree f(x) using its first (g) and second (h) derivatives at the current prediction.
L(y, F+f) approx= L(y,F) + g*f(x) + (1/2)*h*f(x)^2 where g = dL/dF (gradient -- same as GBM residuals) h = d2L/dF2 (Hessian -- curvature of loss)
2
Optimal leaf weight (closed form)
Minimise the Taylor-approximated loss over each leaf. The optimal weight is found analytically — no line search needed.
w*_j = -(sum of g_i in leaf j) / (sum of h_i in leaf j + lambda) # lambda = reg_lambda (L2 penalty on leaf weight) # Larger lambda --> smaller leaf weights --> less overfitting
3
Split gain score
XGBoost evaluates splits using this score. Only split if gain > gamma (pruning threshold).
Gain = 0.5 * [GL^2/(HL+lam) + GR^2/(HR+lam) - (GL+GR)^2/(HL+HR+lam)] - gamma # GL, GR = sum of gradients in left / right child # HL, HR = sum of Hessians in left / right child # gamma = reg_gamma (min gain to make a split)

Feature importance — gain, weight, cover

XGBoost offers three importance types. Gain is most reliable — it measures how much each feature improves the model when used in splits. Weight just counts split frequency (biased for continuous features). Cover averages the number of samples affected by splits on that feature.

Common mistakes

✕
Not using early stopping. Always set n_estimators high (500–1000) and use early_stopping_rounds=30 with a validation set. Without it you’re guessing the right number of trees.
✕
Ignoring tree_method=hist for large data. The default exact method is O(n×d) per node. For >10k rows always set tree_method=’hist’ — same accuracy, 10–100x faster.
✕
Tuning n_estimators manually. n_estimators is best found by early stopping, not grid search. Set it to 1000 and let XGBoost find the right number automatically using your validation set.
✕
Not scaling features. Tree-based models don’t need scaling for splits. But XGBoost’s regularisation (reg_lambda, reg_alpha) works better when features are on similar scales. Always StandardScale.
✕
colsample_bytree=1.0 always. Setting colsample_bytree=0.8 adds Random Forest-style diversity and almost always improves test accuracy on real datasets with many correlated features.

★
Key takeaways

Summary
XGBoost uses second-order gradients (Hessian) to find optimal leaf weights analytically — faster convergence than vanilla GBM.
reg_lambda + reg_alpha + gamma are three independent regularisers. Start with lambda=1, alpha=0, gamma=0 and tune from there.
Always use early_stopping_rounds + a validation set. Set n_estimators=1000 and let XGBoost find the optimal round automatically.
Set tree_method=’hist’ for any dataset >10k rows. It’s faster with the same accuracy.
colsample_bytree=0.8 + subsample=0.8 adds forest-like diversity and consistently improves generalisation on real data.
XGBoost is the go-to for tabular competitions. If time allows, also try LightGBM (leaf-wise, faster on large data) and CatBoost (handles categoricals natively).
