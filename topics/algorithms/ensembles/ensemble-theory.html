Bias-variance decomposition

Every supervised learning error can be broken into three parts. Understanding this decomposition explains exactly why bagging and boosting work in different situations.

→ Bias²
How far the average prediction is from the true answer. High bias = model too simple, underfitting. Linear regression on a curved surface has high bias.
⇅ Variance
How much predictions change across different training datasets. High variance = model too complex, overfitting. Deep trees have high variance.
❖ Noise
Irreducible error from randomness in the data itself. No model can do better than this. You can’t reduce noise — only bias and variance.
Expected Error = Bias² + Variance + Irreducible Noise Bias² = (E[f_hat(x)] - f(x))^2 # how wrong on average Variance = E[(f_hat(x) - E[f_hat(x)])^2] # how unstable

How ensembles reduce error

Different ensemble methods attack different parts of the error decomposition:

Bagging → reduces Variance
Average of B uncorrelated models has variance σ²/B. With 100 trees, variance drops 100×. Works on high-variance models like deep trees.
Var(mean of B models) = sigma^2 / B (when models are uncorrelated) Correlation reduces this benefit -> Random Forest adds feature randomness to reduce correlation
Boosting → reduces Bias
Sequentially correct systematic errors. Each learner is trained to fix what the previous ones got wrong. Can turn a barely-better-than-random stump into a strong classifier.
F_T(x) = sum_t alpha_t * h_t(x) Each h_t corrects residuals from F_{t-1} As T grows, bias shrinks Risk: variance can grow if T too large -> use early stopping

The diversity theorem

The most important result in ensemble theory — the ambiguity decomposition. It proves that diversity always helps, regardless of individual model quality.

Ensemble Error = Average Individual Error - Diversity where: Diversity = (1/B) * sum_i E[(f_i(x) - f_ensemble(x))^2] = avg squared deviation from ensemble mean Implication: 1. More diverse models -> lower ensemble error 2. Can improve ensemble without improving individuals 3. Even adding weak diverse models can help!

Condorcet’s Jury Theorem — why majority voting works

From 18th-century political science: if each voter has a better-than-random chance of being correct, and votes are independent, then majority vote approaches 100% accuracy as the jury size grows.

If each classifier has accuracy p > 0.5 and errors are independent: P(ensemble correct) = sum_{k=ceil(n/2)}^{n} C(n,k) * p^k * (1-p)^(n-k) As n -> infinity, P(correct) -> 1.0 Example: 21 classifiers each at 70% accuracy P(majority correct) = ~97.6% Caveats: - Errors must be UNCORRELATED (hard to guarantee) - Each model must be > 50% (also not guaranteed for all inputs) - This is an upper bound; real gains are smaller

Key takeaways

Summary
Error = Bias² + Variance + Noise. Bagging attacks Variance. Boosting attacks Bias. You can’t reduce noise.
Diversity theorem: Ensemble Error = Avg Individual Error − Diversity. More diverse models always produce a lower ensemble error.
Condorcet’s theorem: If models are better than random and errors are independent, majority vote accuracy approaches 100% as n grows. The independence assumption is the hard part.
Variance of an average = σ²/B for uncorrelated models. Correlation between models limits the reduction — which is why Random Forest adds extra randomness.
Rule of thumb: Start with Random Forest (best single model for tabular data). Add GBM/XGBoost for more accuracy. Use stacking only when you need maximum performance and have enough data.
