Gradient Boosting

Build trees one at a time, each one correcting the mistakes of all the trees before it. Instead of averaging independent trees like Random Forest, gradient boosting trains trees sequentially — each new tree fits the residual errors left by the previous ensemble.

⏱ 25 min ◆ Intermediate numpy · sklearn · XGBoost

💡
The core idea — learn from your mistakes

Imagine you are trying to predict house prices. Your first simple model predicts $200k for everything. Some houses are $350k — the error is $150k. Some are $180k — the error is -$20k. Instead of training a brand new model, you train a second tree to predict those errors. Then a third tree to predict the errors of the first two combined. And so on.

This is gradient boosting: each new tree is a small correction added to the running total. After 100 trees, each making tiny corrections, the ensemble prediction is very accurate.

Key difference from Random Forest: Random Forest trains all trees independently and averages them (parallel). Gradient Boosting trains each tree on the errors of the previous ones (sequential). Sequential = slower training but usually better accuracy.

Learning rate × n_estimators — the shrinkage tradeoff

The learning rate and number of trees are inversely linked. Low learning rate needs more trees. High learning rate converges faster but risks overfitting. The best practice is: lower the learning rate and increase trees until test error stops improving.

Effect of learning rate on convergence speed
⚠

Golden rule: Set learning_rate=0.05 or 0.1, then find the best n_estimators using early stopping. This always outperforms using a large learning rate with fewer trees. Use early_stopping_rounds=10 in XGBoost/sklearn to stop automatically.

∑
The algorithm — step by step (beginners welcome)

1
Start with a constant prediction
The first prediction is simply the mean of all y values (for regression) or the log-odds (for classification). This is our starting point.
F₀(x) = mean(y) # for regression F₀(x) = log(p/(1-p)) # for classification
2
Compute the residuals (errors)
For each training sample, calculate the difference between the actual value and the current prediction. These residuals are what the next tree will learn to predict.
residual_i = y_i - F(x_i) # for MSE loss # (for other losses, we use the negative gradient instead)
3
Train a small tree on the residuals
A shallow decision tree (max_depth=3 is typical) is fitted to predict the residuals. This tree only needs to reduce the remaining error, not model the full problem.
T_m = DecisionTree().fit(X, residuals)
4
Update the ensemble prediction
Add the new tree's predictions to the running total, scaled by the learning rate α. Small α means each tree contributes only a little — this is the "shrinkage" regularisation.
F_m(x) = F_{m-1}(x) + alpha * T_m(x) # alpha = learning_rate (typically 0.01 to 0.3)
5
Repeat until n_estimators trees are built
Go back to step 2 with the updated predictions. Each iteration reduces the residuals further. Stop when you hit n_estimators trees, or use early stopping when validation loss stops improving.
for m in range(1, n_estimators + 1): residuals = y - F_{m-1}(x) T_m = fit_tree(X, residuals) F_m = F_{m-1} + learning_rate * T_m

Common mistakes

✕
Using default learning_rate=0.1 without tuning n_estimators. Learning rate and n_estimators must be tuned together. Low LR needs more trees. Always use early stopping or cross-validate both simultaneously.
✕
Not scaling features when using sklearn GBM. Unlike tree-based Random Forest, GBM with certain loss functions can be sensitive to outliers. Robust scalers or log transforms can help, especially for regression.
✕
Ignoring overfitting signals. GB can achieve 100% training accuracy while performing poorly on test data. Always plot both training and validation loss curves. Stop when validation stops improving.
✕
Using max_depth > 5. Gradient Boosting works best with shallow trees (depth 1-5). Deep trees make each learner too powerful — the ensemble overfits. Start with max_depth=3.
✕
Not using XGBoost/LightGBM for large datasets. sklearn's GradientBoostingClassifier is slow on large datasets. Switch to XGBoost or LightGBM for 10x-100x speedup. They also handle missing values natively.

★
Key takeaways

Summary
Gradient Boosting builds trees sequentially. Each tree fits the residual errors of the previous ensemble, not the original target.
learning_rate scales each tree's contribution. Low rate + more trees = better generalisation. This is called shrinkage.
Use shallow trees (max_depth=3) as the base learners. Deep trees overfit. GBM corrects errors across many trees, not within one deep tree.
GBM typically outperforms Random Forest on tabular data but needs more careful tuning and is slower to train.
For production use, prefer XGBoost or LightGBM over sklearn's GBM — they are much faster and handle missing data natively.
Always use early stopping — monitor validation loss and stop when it stops improving to prevent overfitting automatically.
