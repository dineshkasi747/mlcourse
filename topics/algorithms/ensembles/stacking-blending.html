Stacking & Blending

Voting says "average the predictions." Stacking says "train a new model to learn how to combine the predictions." Instead of a fixed rule (majority vote), a meta-learner figures out which base models to trust and when.

⌛ 20 min ◆ Intermediate sklearn

🌟
Intuition — a manager who learned from the experts

In voting, you treat all models equally (or assign fixed weights). In stacking, you add a smarter step: you train a meta-learner whose only job is to look at what all the base models predicted and decide the final answer.

The meta-learner learns patterns like: "When the Decision Tree and SVM agree, trust them. When they disagree, trust the Logistic Regression." This is much smarter than simple averaging.

💡

Simple analogy: Five weather forecasters each give a prediction. Voting says "take the average." Stacking says "hire a chief meteorologist who has studied how accurate each forecaster is in different conditions, and let them decide." The chief meteorologist is the meta-learner.

Stacking vs Blending — what is the difference?

Both approaches train base models and then combine them with a meta-learner. The difference is how you create the training data for the meta-learner.

Stacking — cross-validation
1. Split training data into K folds
2. For each fold: train on other folds, predict this fold
3. Build meta-features = out-of-fold predictions
4. No data wasted, but slower
Uses ALL training data. No data leak. Preferred approach.
Blending — holdout set
1. Split: 70% train / 20% blend / 10% test
2. Train base models on 70%
3. Predict the 20% blend set
4. Train meta-learner on these predictions
Simpler to implement. Wastes some data. Good first try.

How stacking works — step by step

1
Train base models with cross-validation
Use K-fold cross-validation. For each fold, train on K-1 folds, predict the held-out fold. This gives you "out-of-fold" predictions for every training sample without data leakage.
For each fold k in [1..K]: base_model.fit( X_train[folds != k] ) oof_preds[folds == k] = base_model.predict( X_train[folds == k] ) Result: oof_preds has predictions for EVERY training sample But each prediction was made on unseen data (no leakage!)
2
Build the meta-feature matrix
Stack the out-of-fold predictions from all base models side by side. Each column is one base model's predictions. This new matrix becomes the training data for the meta-learner.
For 3 base models, meta-features look like: [ DT_pred | LR_pred | KNN_pred ] <- new feature matrix [ 0.72 | 0.81 | 0.65 ] <- sample 1 [ 0.23 | 0.19 | 0.31 ] <- sample 2 [ 0.89 | 0.95 | 0.77 ] <- sample 3 ... The meta-learner trains on this and learns which model to trust
3
Train the meta-learner
Train any model (usually Logistic Regression or Ridge) on the meta-features to predict y. The meta-learner's coefficients tell you how much it weights each base model.
meta_model.fit( meta_features, y_train ) # Meta-learner might learn: # final_pred = 0.2 * DT_pred + 0.5 * LR_pred + 0.3 * KNN_pred # (these weights are LEARNED, not hand-set like in voting)
4
Making predictions on new data
For a new test point: get predictions from all base models, assemble the meta-feature row, feed it to the meta-learner. The meta-learner gives the final prediction.
For a new test sample x_test: 1. dt_pred = dt_model.predict(x_test) = 0.68 2. lr_pred = lr_model.predict(x_test) = 0.82 3. knn_pred = knn_model.predict(x_test) = 0.71 meta_row = [0.68, 0.82, 0.71] final = meta_model.predict(meta_row) = Class A

Common mistakes

✕
Data leakage in stacking. The biggest pitfall. If you train base models on the full training set and then predict the same training set for meta-features, the meta-learner sees "perfect" predictions and massively overfits. Always use out-of-fold predictions (cross-validation) or a separate holdout set.
✕
Using a complex meta-learner. The meta-learner should be simple (Logistic Regression, Ridge). A complex meta-learner (like a deep tree) will overfit the meta-features. Keep it simple.
✕
Not using diverse base models. If all base models are Decision Trees, stacking won't help. You need different model types — trees, linear models, KNN, neural nets — that make different errors.
✕
Over-engineering on small datasets. Stacking needs enough data for base models to generalise AND for the meta-learner to learn. With less than ~500 samples, simple cross-validated models often beat stacking.

★
Key takeaways

Summary
Stacking trains a meta-learner to combine base model predictions — instead of using fixed rules like majority vote or averaging.
Use out-of-fold predictions (cross-validation) to build meta-features. This prevents data leakage. Never predict on the same data you trained on.
The meta-learner should be simple — Logistic Regression or Ridge are standard choices.
Blending is the simpler version: hold out 20% of data, train base models on 80%, use the 20% predictions as meta-features. Faster to implement but wastes data.
sklearn's StackingClassifier and StackingRegressor handle the cross-validation and meta-training automatically.
