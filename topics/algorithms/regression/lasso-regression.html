Lasso Regression (L1)

Ridge shrinks coefficients. Lasso kills them outright. By penalising the sum of absolute values instead of squares, Lasso drives irrelevant feature coefficients to exactly zero — automatically selecting only the features that matter.

⏱ 20 min ◆ Intermediate numpy · sklearn

✂
Intuition — the automatic feature eraser

Imagine you have 50 features but only 5 actually predict the target. Ridge will shrink all 50 coefficients a little — you still have 50 features in your model. Lasso will shrink 45 of them all the way to exactly zero, leaving you with a clean 5-feature model that's easy to interpret and deploy.

This is the single most important difference between Ridge and Lasso: Lasso does automatic feature selection. It's not just regularisation — it's a built-in variable selector.

🧠

Why does L1 produce zeros but L2 doesn't? The L1 penalty (|β|) has a sharp corner at zero — the gradient jumps from −1 to +1 right at β=0. This corner is like a trap: once a coefficient gets close to zero, the penalty actively pulls it to exactly zero and keeps it there. The L2 penalty (β²) has a smooth curve — its gradient near zero is nearly flat, so coefficients just get pushed close to zero but never quite reach it.

Geometry — the diamond creates exact zeros

This is the most important visualisation in all of regularisation. Ridge uses a circle (L2 ball). Lasso uses a diamond (L1 ball). The diamond has sharp corners that sit exactly on the axes. The loss contours (ellipses) almost always hit the diamond at a corner — right where one coefficient is exactly zero.

Key insight for beginners: A circle has no corners — when you shrink it, it's almost impossible for the minimum of an arbitrary loss function to land exactly on an axis. A diamond has sharp corners that sit exactly on the axes (where β=0). Shrinking the diamond almost always makes the solution snap to a corner — giving you an exact zero.

↕
Coordinate descent — how Lasso is actually solved

Unlike Ridge which has a neat closed-form formula, Lasso has no closed-form solution because |β| is not differentiable at zero. The standard solver is coordinate descent: optimise one coefficient at a time, holding all others fixed, cycling through all features until convergence.

Each single-feature update has a clean analytical solution called soft thresholding: if the OLS coefficient for that feature would be smaller than the penalty, set it to zero. Otherwise, shrink it toward zero by the penalty amount.

Hyperparameter tuning — alpha controls sparsity

Lasso's alpha controls how many features survive. Small alpha = nearly OLS (all features kept). Large alpha = very sparse (few features kept). Use LassoCV to find the alpha that gives the best cross-validated score while achieving useful sparsity.

The math — L1 penalty and soft thresholding

1
Lasso objective function
Replace Ridge's squared penalty (||β||²) with absolute value penalty (||β||₁). This small change causes dramatically different behaviour — exact zeros become possible.
Lasso loss = ||y - Xβ||² + alpha · ||β||₁ fitting error L1 penalty = sum of |β_j| vs Ridge: ||y - Xβ||² + alpha · ||β||² L2 penalty = sum of β_j²
2
Why no closed form? The kink at zero
|β| is not differentiable at β=0. You can't just set the gradient to zero and solve like OLS/Ridge. Instead, use the subdifferential (generalised gradient). The solution exists but requires an iterative algorithm.
d/dβ |β| = { -1 if β < 0 { +1 if β > 0 { [-1, +1] if β = 0 ← not a single value! This "kink" is why L1 creates exact zeros: the subdifferential can equal zero at β=0 exactly.
3
Coordinate descent update — soft thresholding
Update one coefficient at a time. For feature j, compute the OLS residual ignoring j (the "partial residual"), then apply soft thresholding: if the raw estimate is smaller than alpha/2, set to zero. Otherwise shrink by alpha/2.
Partial residual: r_j = y - X_{-j} β_{-j} OLS update: rho_j = X_j · r_j / ||X_j||² Soft threshold: β_j = S(rho_j, alpha/(2||X_j||²)) where S(z, t) = sign(z) · max(|z| - t, 0) = 0 if |z| <= t (zero out small coefficients!) = z-t if z > t (shrink positive coefficient) = z+t if z < -t (shrink negative coefficient)
4
Why soft thresholding creates exact zeros
If the "signal" for feature j (the partial correlation with residuals) is smaller than the penalty threshold, the feature is zeroed out completely. The penalty is strong enough to overcome the signal entirely.
If |X_j · r_j| < alpha/2: set β_j = 0 (feature eliminated!) Intuition: feature j explains less variance than it costs in penalty — not worth keeping in the model. Ridge: β_j = rho_j / (1 + alpha) ← always nonzero Lasso: β_j = S(rho_j, alpha/2) ← zero when signal weak

Common mistakes

✕
Not scaling before Lasso. Just like Ridge, Lasso penalises coefficient size — but a feature in large units will have a tiny coefficient and escape the penalty unfairly. Always StandardScaler before Lasso.
✕
Using Lasso when features are correlated. With correlated features, Lasso arbitrarily keeps one and zeros the other. The kept feature depends on tiny data fluctuations. Use ElasticNet (ridge + lasso combined) for correlated features.
✕
Trusting zeroed features are truly irrelevant. With a small dataset, Lasso may zero out a feature that matters. Always validate feature selection stability with bootstrap resampling or stability selection.
✕
Setting max_iter too low. Lasso uses coordinate descent which can need many iterations on large datasets. If you see a ConvergenceWarning, increase max_iter (default 1000, try 10000) or increase tol.
✕
Forgetting that alpha in sklearn is alpha/(2n). sklearn's Lasso uses loss = (1/2n)||y-Xβ||² + alpha||β||₁. The 1/n scaling means optimal alpha depends on dataset size. LassoCV handles this correctly.

★
Key takeaways

Summary
Lasso adds alpha · ||β||₁ (sum of absolute values). This L1 penalty creates exact zeros — automatic feature selection.
The diamond shape of the L1 ball is why zeros happen: its corners sit exactly on the axes, and loss contours almost always hit a corner first.
Lasso is solved with coordinate descent + soft thresholding. If a feature's signal is weaker than the penalty, it gets zeroed out completely.
Use Lasso when you believe few features truly matter out of many. Use Ridge when all features contribute a little. Use ElasticNet for both.
Always scale features first. Use LassoCV for automatic alpha selection. Watch for ConvergenceWarning and increase max_iter if needed.
With correlated features, Lasso unstably picks one and drops the other. ElasticNet (next topic) fixes this by combining L1 + L2.
