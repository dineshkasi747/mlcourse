1
Initialise parameters
Set weights w and bias b to small random values. The starting point determines the path, not the destination (on convex surfaces).
w = 0.0     b = 0.0
2
Forward pass - compute predictions
Multiply inputs by weights and add bias to get predictions y-hat.
y_hat = X*w + b
3
Compute the loss
Measure how wrong the predictions are. MSE squares the errors making it differentiable everywhere.
L = (1/n) * sum( (y_hat - y)^2 )
4
Compute gradients
Differentiate the loss with respect to each parameter. Gradient points uphill - go the opposite direction.
dL/dw = (2/n)*X^T*(y_hat-y)    dL/db = (2/n)*sum(y_hat-y)
5
Update parameters
Subtract alpha times the gradient. Alpha controls step size.
w = w - alpha * dL/dw     b = b - alpha * dL/db
6
Repeat until convergence
Go back to step 2. Stop when loss stops decreasing or max iterations reached.
while |L_prev - L_curr| > eps: repeat steps 2-5

Batch vs SGD vs Mini-Batch

Click a variant to see its path. Notice how noise in SGD helps it escape local minima that trap Batch GD.

Batch GD
All data per step
Uses entire dataset for each gradient update. Smooth and accurate but slow on large data.
+ Smooth convergence
+ Exact gradient
- Slow on big data
- High memory
Stochastic GD
1 sample per step
Updates after every single sample. Noisy but fast - noise helps escape local minima.
+ Very fast updates
+ Escapes local minima
- Very noisy path
- Oscillates at end
Mini-Batch GD
32-256 per step
Best of both worlds. Industry standard used by PyTorch and TensorFlow by default.
+ Fast and stable
+ GPU-efficient
- Batch size to tune
- Still some noise

Common mistakes

x
Learning rate too high causes divergence. Loss explodes to NaN. Start with alpha=0.01 and halve if loss increases.
x
Not normalising features. If X1 is 0-1 and X2 is 0-10000, gradients are unequal. Always StandardScale before GD.
x
Forgetting to shuffle for SGD. Sorted data means batches are all one class - gradients are biased.
x
Stopping too early. Loss may plateau briefly then drop again. Use patience-based early stopping.
x
Same LR for all parameters. Use Adam or RMSProp - they adapt the LR per parameter automatically.

Key takeaways

Summary
GD moves parameters in the direction of steepest descent until loss converges.
Learning rate alpha is the most critical hyperparameter - too small is slow, too large diverges.
Always normalise features before running gradient descent.
Mini-batch GD (32-256) is the industry standard used in all neural networks.
In practice use Adam - it adapts the learning rate per parameter automatically.
Monitor the loss curve: smooth decrease is healthy, spikes mean LR too high.
