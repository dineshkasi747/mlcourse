Intuition — what does each piece mean?

↗
Slope (w)
For every 1-unit increase in X, y increases by w units. If w = 2.5: add 1 year of experience, salary goes up $2500. The rate of change.
•
Intercept (b)
The predicted y when X = 0. Often has no real-world meaning (age = 0 is meaningless for salary), but it anchors the line vertically.
↔
Residuals
The vertical distance from each point to the line: e = y - y_hat. OLS minimises the sum of squared residuals. Positive = above line, negative = below.
R²
R² score
Fraction of variance explained by the model. R² = 1 means perfect fit. R² = 0 means the model is no better than predicting the mean. R² < 0 means worse than mean.
✓
OLS solution
The optimal w and b are found analytically — no iterations needed. w = cov(X,y)/var(X) and b = mean(y) - w * mean(X). This is exact, not approximate.
⚠
Assumptions
OLS assumes the relationship is truly linear, residuals are normally distributed, and there’s no heteroscedasticity. Violating these makes estimates unreliable.

4
OLS derivation — step by step

These four steps show exactly how the closed-form formulas for w and b are derived from the loss function.

1
The model
Define the prediction equation
A straight line with slope w and intercept b. We want to find the w and b that make predictions y_hat as close as possible to the true values y.
y_hat = w * x + b
2
The loss
Mean Squared Error loss function
MSE squares each error so all residuals are positive and large errors are penalised more. We want to minimise this over all n training samples.
L(w, b) = (1/n) * sum_i (y_hat_i - y_i)^2 = (1/n) * sum_i (w*x_i + b - y_i)^2
3
Calculus
Set partial derivatives to zero
At the minimum of L, both partial derivatives must be zero. Solving these two equations simultaneously gives the exact optimal w* and b*.
dL/dw = (2/n) * sum_i x_i*(w*x_i + b - y_i) = 0 dL/db = (2/n) * sum_i (w*x_i + b - y_i) = 0
4
Closed-form solution
OLS formulas for w* and b*
Solving the system gives the exact optimal parameters. Note: b* depends on w*, so always compute w* first. These formulas work for any dataset in one pass — no iterations, no learning rate.
w* = cov(X, y) / var(X) = sum_i (x_i - x_mean)(y_i - y_mean) ----------------------------------- sum_i (x_i - x_mean)^2 b* = y_mean - w* * x_mean

MAE — Mean Absolute Error interpretable
MAE = (1/n) * sum |y_i - y_hat_i|
Average absolute error in the same units as y. If predicting house prices in $, MAE = $15,000 means you are on average $15K off. Easy to explain to non-technical stakeholders.
MSE — Mean Squared Error differentiable
MSE = (1/n) * sum (y_i - y_hat_i)^2
Squares errors, so large errors are penalised heavily. Units are squared (e.g. $^2), making it uninterpretable directly. Used as the OLS loss function because it is differentiable.
RMSE — Root MSE most common
RMSE = sqrt(MSE)
Square root of MSE, restoring original units. Penalises large errors more than MAE but is interpretable. The most commonly reported regression metric in research papers.
R² — Coefficient of Determination relative
R2 = 1 - SS_res / SS_tot SS_res = sum (y - y_hat)^2 SS_tot = sum (y - y_mean)^2
Proportion of variance explained. 0.85 means the model explains 85% of the variance in y. Scale-free, so comparable across datasets. Can be negative if the model is worse than predicting the mean.

When does linear regression fail?

OLS has four key assumptions. Click each to see a canvas simulation of what violation looks like.

1
Linearity
The relationship between X and y must be linear. If it is curved, linear regression will systematically underfit. Use polynomial features or a non-linear model.
2
Normal residuals
Residuals should be normally distributed around zero. Skewed residuals mean the model is systematically wrong in some regions. Check with a Q-Q plot.
3
Homoscedasticity
Variance of residuals must be constant across all X values (homoscedastic). If variance grows with X (heteroscedastic), confidence intervals and p-values are invalid.
4
Independence
Observations must be independent. Time series data often violates this (autocorrelation). Use the Durbin-Watson test to detect autocorrelated residuals.

Common mistakes

x
Using R² alone to judge model quality. R² = 0.95 sounds great, but if the residuals are non-random (curved pattern), the model is still wrong. Always plot residuals vs fitted values.
x
Fitting without checking linearity. If the scatter plot shows a curve, linear regression will give biased predictions at both ends. Add polynomial features or use a non-linear model.
x
Extrapolating beyond the training range. Linear regression assumes the relationship continues linearly forever. Predicting house price for 10,000 sq ft when max training was 4,000 sq ft is unreliable.
x
Ignoring influential outliers. One outlier can pull the slope significantly. Always check Cook’s distance to identify high-leverage points before trusting the model.
x
Confusing correlation with causation. A high R² does not mean X causes y. Ice cream sales and drowning deaths are correlated (both increase in summer) but neither causes the other.

Key takeaways

Summary
Simple linear regression fits a line y = wx + b by minimising the sum of squared residuals (OLS).
OLS has a closed-form solution: w* = cov(X,y)/var(X) and b* = mean(y) - w* * mean(X). No iterations needed.
Always check the four OLS assumptions: linearity, normal residuals, homoscedasticity, independence.
Report RMSE (interpretable, same units as y) and R² (variance explained). Never use R² alone.
Plot residuals vs fitted values. A random horizontal band means the model is well-specified. Any pattern means the model is wrong.
SLR is the foundation: Multiple LR, Ridge, Lasso, and Polynomial Regression all extend this same framework.
