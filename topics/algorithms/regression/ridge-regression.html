Ridge Regression (L2)

OLS finds the best fitting line. Ridge finds the best fitting line that is also not too wild. It adds a penalty for large coefficients — shrinking them toward zero, reducing overfitting, and fixing the multicollinearity problem.

⏱ 20 min ◆ Intermediate numpy · sklearn

🏔
Intuition — taming wild coefficients

Imagine you're fitting a high-degree polynomial and the model assigns coefficient = +5000 to x⁴ and −4998 to x³ — these cancel each other out and barely help, but together they cause wild oscillations on new data.

Ridge says: "You can still fit the data well, but I'm going to charge you a fee for every unit of coefficient size." The model learns to use smaller, more stable coefficients that generalise better — even if training error goes up slightly.

🧠

Everyday analogy: OLS is a student who memorises every answer from past exams. Ridge is a student who actually understands the material — their score on old exams is slightly lower, but they ace new exams because they learned the real pattern, not the noise.

Geometry — a circle constrains the solution

The most important visual in all of regularisation: OLS finds the point that minimises squared error — it can be anywhere. Ridge adds a circle constraint around the origin (L2 ball). The solution must land inside that circle — so it gets pulled toward zero. Drag the α slider below and watch the circle shrink.

OLS vs Ridge — constraint geometry
α: 0.50
Red dot = OLS solution (unconstrained) · Purple dot = Ridge solution (constrained to circle) · Ellipses = contours of equal loss · As α↑ circle shrinks → solution pulled closer to origin
OLS (α = 0)
Minimises only fitting error. Coefficients can be huge. Unstable with correlated features. Can overfit badly.
Ridge (α > 0)
Minimises fitting error + α·||β||². Shrinks ALL coefficients toward zero. Never exactly zero. Stable with correlated features.
Key difference from Lasso
Ridge uses a circle (L2). Lasso uses a diamond (L1). Circle corners don't touch axes → Ridge never sets coefficients to exactly zero.

Regularisation path — how each coefficient shrinks

As α increases from 0 (OLS) to ∞ (all coefficients → 0), each coefficient follows its own shrinkage path. Features with strong signal resist longer before shrinking. Features that are noise shrink away quickly. This is the Ridge path.

Hyperparameter tuning — finding the best α

α is Ridge's only hyperparameter. Too small = still overfitting. Too large = underfitting (all coefficients crushed to near-zero). The sweet spot is found with cross-validation. RidgeCV does this automatically.

The math — adding an L2 penalty

1
Ridge objective function
Add a penalty term to OLS loss. The penalty is the sum of squared coefficients (L2 norm) multiplied by α. Note: the intercept β₀ is NOT penalised.
Ridge loss = ||y - Xβ||² + α·||β||² ↑ ↑ fitting error regularisation (penalise big β) α = 0 → plain OLS α → ∞ → β → 0 (all coefficients shrink to zero)
2
Closed-form solution
Take gradient, set to zero — same as OLS Normal Equations but with α added to the diagonal of XᵀX. This guarantees XᵀX + αI is always invertible (no multicollinearity singularity).
∂L/∂β = -2Xᵀ(y - Xβ) + 2αβ = 0 β̂_ridge = (XᵀX + αI)⁻¹ Xᵀy ↑ α·I "inflates" diagonal → always invertible even when features are perfectly correlated!
3
Bias-Variance tradeoff in Ridge
Ridge introduces bias (coefficients pulled away from true values) but reduces variance (stable coefficients). Total error = Bias² + Variance + irreducible noise. Optimal α balances this.
OLS: Low bias, High variance (bad when p is large) Ridge: Some bias, Low variance (good generalisation) Bias of β̂_ridge = -α(XᵀX + αI)⁻¹ β_true Var of β̂_ridge = σ²(XᵀX + αI)⁻¹XᵀX(XᵀX + αI)⁻¹ As α ↑: Bias ↑ but Variance ↓ → sweet spot exists
4
Ridge vs OLS in singular cases
When features are perfectly correlated, XᵀX is singular (non-invertible) and OLS fails completely. Ridge adds α to the diagonal, making inversion always possible. This is Ridge's most important practical benefit.
If X₁ = X₂ (perfect collinearity): XᵀX is singular → OLS fails (undefined) XᵀX + αI is non-singular → Ridge works! Ridge effectively "spreads" the credit between correlated features equally and stably.

Common mistakes

✕
Not scaling features before Ridge. Ridge penalises large coefficients, but large coefficients can come from small-scale features (e.g. x in millimetres → coefficient is tiny). StandardScaler first — always.
✕
Penalising the intercept. The intercept β₀ should NOT be regularised. sklearn's Ridge sets fit_intercept=True by default (correct). If you implement Ridge from scratch, subtract the mean from X and y before fitting, then add back the intercept.
✕
Using Ridge when you need feature selection. Ridge shrinks coefficients but never sets them to exactly zero. If you want automatic feature selection, use Lasso (L1 penalty). Ridge keeps all features in the model.
✕
Picking α without cross-validation. Never set α by hand. Use RidgeCV which sweeps a range of α values and picks the best by cross-validation. It's fast (uses analytical trick) and always correct.
✕
Expecting Ridge to fix non-linearity. Ridge only regularises OLS. If the relationship is non-linear, Ridge still won't fit it. Combine PolynomialFeatures + Ridge for curved data.

★
Key takeaways

Summary
Ridge adds α·||β||² to the OLS loss. Solution: β̂ = (XᵀX + αI)⁻¹Xᵀy. Always invertible, always stable.
Coefficients are shrunk toward zero but never exactly zero. Use Lasso if you need exact zeros (feature selection).
Always scale features first. StandardScaler → Ridge. Without scaling, the penalty treats features unfairly.
Ridge fixes multicollinearity completely. OLS fails when features are correlated; Ridge is stable by design.
Use RidgeCV to auto-select α. It tries many values and picks the best using efficient leave-one-out cross-validation.
Ridge = OLS + L2 ball constraint. The L2 ball is a circle in 2D. Lasso uses an L1 ball which is a diamond — corners touch axes → exact zeros.
