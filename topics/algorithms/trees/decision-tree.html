Decision Tree

A model that learns a sequence of if-then rules from data. At each node it asks one question about one feature, splitting data into purer subsets until reaching a decision. Fully interpretable — you can read every rule.

⏱ 22 min ◆ Beginner numpy · sklearn

🌿
How a decision tree thinks

A decision tree asks one yes/no question at a time, following the answer to the next question until it reaches a final prediction. It's the same logic you use when diagnosing a problem — "Is the screen on? → Is there WiFi? → Is the battery full?"

The algorithm's job is to figure out which question to ask first, which question to ask second, and so on — choosing the split at each step that makes the resulting groups as pure (one-class) as possible. That purity is measured by Gini impurity or entropy.

Key insight: Trees make no assumptions about feature distributions (unlike Naive Bayes or Logistic Regression). They work on any mixture of numeric and categorical features, require no scaling, and produce rules you can read and explain to a non-technical audience.

Gini vs Entropy — split criteria visualised

At each node, the tree evaluates every possible split on every feature and picks the one that maximises Information Gain — the reduction in impurity after the split.

Gini Impurity
Gini = 1 - Σ pᵢ²
Measures probability of misclassifying a randomly chosen element. Faster to compute. Default in sklearn.
Entropy (Info Gain)
H = -Σ pᵢ log₂(pᵢ)
Measures information content. Slightly more expensive but considers log scale — can prefer more balanced splits.
MSE (Regression)
MSE = (1/n)Σ(yᵢ-ȳ)²
For regression trees. Splits minimise variance of target values in each child node.

Feature importance — what the tree learned

After training, each feature gets an importance score: the total reduction in impurity it caused across all splits, weighted by the number of samples at each split. Features used near the root matter more because they affect more samples.

The CART algorithm — step by step

sklearn uses CART (Classification and Regression Trees). Here's exactly what it does at each node:

1
Try every feature × every threshold
For each feature j and each unique value t in that feature, consider splitting: left = {x: xⱼ ≤ t}, right = {x: xⱼ > t}. This is O(n·d) per node.
Candidates: (j, t) for j ∈ features, t ∈ unique_values(Xⱼ)
2
Compute impurity before and after each split
Parent impurity minus weighted average child impurity = Information Gain. We want to maximise this.
IG(j,t) = Impurity(parent) - [nL/n · Impurity(left) + nR/n · Impurity(right)]
3
Choose the best split
The split (j*, t*) with maximum information gain is applied. The node stores: feature index j*, threshold t*, and class counts for prediction.
(j*, t*) = argmax_{j,t} IG(j, t)
4
Recurse on children
Apply the same algorithm to each child node. Stop when: max_depth reached, fewer than min_samples_split samples, or node is already pure (impurity = 0).
Stopping conditions: depth >= max_depth OR n < min_samples_split OR Impurity = 0 OR all features have same value
5
Predict: traverse from root to leaf
Follow the tree: at each internal node, go left if xⱼ ≤ t, right otherwise. At the leaf, return majority class (classification) or mean y (regression).
predict(x): node = root while node is not leaf: if x[node.feature] <= node.threshold: node = node.left else: node = node.right return node.majority_class

Pruning — fighting overfitting

An unpruned tree will memorise training data — it can achieve 100% train accuracy by creating a separate leaf for each training sample. Pruning removes branches that don't improve generalisation.

Pruning effect — ccp_alpha sweep
Higher ccp_alpha → more pruning → fewer nodes → lower train accuracy but often higher test accuracy

Common mistakes

✕
Leaving max_depth=None. An unlimited tree will perfectly fit training data (100% train accuracy) but catastrophically overfit. Always set max_depth or use ccp_alpha pruning. Check train vs test gap.
✕
Trusting feature importances blindly. Importance scores are biased toward high-cardinality features (features with many unique values get more split opportunities). Use permutation importance or SHAP for unbiased estimates.
✕
Using a single tree in production. A single tree has high variance — small data changes can completely restructure it. Use Random Forest or Gradient Boosting instead, which average many trees.
✕
Not checking for class imbalance. Impurity metrics assume balanced classes. With 95% class A, a pure class-A leaf looks great even though it's useless. Set class_weight='balanced' or use balanced sampling.
✕
Axis-aligned splits miss diagonal boundaries. Decision trees can only split parallel to feature axes. A diagonal boundary requires exponentially many splits. For diagonal or circular patterns, use kernel SVM or ensemble methods.

Key takeaways

Summary
A decision tree greedily finds the best split at each node using Gini impurity or entropy, recursing until stopping conditions are met.
max_depth is the most important hyperparameter. Small = underfit, large = overfit. Tune it with cross-validation and plot the train/test gap.
No feature scaling needed. Decision trees are invariant to monotonic transformations of features.
Fully interpretable — you can export and read every rule. Use export_text() or plot_tree() to visualise.
Single trees have high variance. For production, use Random Forest (bagging) or XGBoost (boosting) — both build on decision trees.
ccp_alpha pruning is the cleanest regularisation method. Sweep alpha values and pick where validation accuracy peaks.
