Data types — what pandas infers and why it matters

After loading, always check df.dtypes. Pandas sometimes guesses wrong — a column of years becomes float64 instead of int, or a boolean column becomes object. Fixing dtypes saves memory and prevents silent errors in your ML pipeline.

Common mistakes

✕
Not checking dtypes after loading. Always run df.info() immediately. A column you expect to be numeric might be object (string) if it has a single bad value like "N/A". This silently breaks all math on that column.
✕
Ignoring encoding errors. Files from Windows often use latin-1 or cp1252 encoding. If you get a UnicodeDecodeError, try encoding='latin-1' or encoding_errors='replace'.
✕
Loading huge files all at once. For files larger than your RAM, use chunksize=10000 to process in chunks, or use usecols=[...] to load only the columns you need.
✕
Assuming index_col=0 is always right. Some CSVs have a saved DataFrame index as the first column (called "Unnamed: 0"). Use index_col=0 in that case. For regular CSVs, leave it as default.
✕
Not handling missing value strings. Your CSV might use "NA", "N/A", "null", "None", "-", or "" for missing data. Pass na_values=['NA','N/A','null','None','-'] to catch them all.

★
Key takeaways

Summary
The basic call is just df = pd.read_csv("file.csv"). Add parameters only when you have a specific problem to solve.
Always run df.info() and df.head() after loading. Check shapes, dtypes, and missing value counts before anything else.
Use sep=';' or sep='\t' for non-comma delimiters. Use sep=None, engine='python' to auto-detect.
Use dtype={'col': str} to prevent pandas from guessing wrong types. Use parse_dates=['date_col'] for datetime columns.
For large files use usecols to load only needed columns, and chunksize to process in batches without filling RAM.
