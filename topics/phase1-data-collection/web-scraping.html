Web Scraping

Most real-world data lives on websites — not in neat CSV files. Web scraping lets you automatically extract data from web pages by parsing their HTML. It is one of the most essential data collection skills in ML.

⌛ 20 min ◆ Beginner requests · BeautifulSoup · pandas

🌎
What is web scraping?

Every web page is just an HTML text file. Your browser reads that file and renders it visually. Web scraping means writing code to read that same HTML file and pull out the specific data you want — prices, headlines, reviews, stock data, weather, anything.

The two most common Python tools are:

  • requests — downloads the HTML content of a URL (like a browser would)
  • BeautifulSoup — parses that HTML and lets you search for tags, classes, IDs to extract specific data

Understanding HTML — what you are parsing

Before scraping, you need to understand the HTML structure. HTML is a tree of tags (like <div>, <p>, <a>). Each tag can have attributes (like class, id, href). BeautifulSoup lets you find tags by name, class, id, or CSS selectors.

Ethics and rules — always check before you scrape

⚠ Always follow these rules
  • Check robots.txt (e.g. example.com/robots.txt). If a path is Disallowed, don't scrape it.
  • Check the site's Terms of Service. Many sites explicitly forbid scraping.
  • Add delays between requests (time.sleep(1–3)). Don't hammer servers.
  • Identify yourself with a proper User-Agent header so the server knows who is making requests.
  • Prefer official APIs when they exist — they are faster, more reliable, and always legal.
  • Never scrape personal data (names, emails, addresses) without explicit permission — GDPR applies.

Step-by-step — the complete scraping workflow

1
Send an HTTP request to get the HTML
Use requests.get() to download the page. Always check the status code — 200 means OK, 404 means not found, 403 means forbidden. Set a User-Agent header so you look like a browser.
import requests headers = {'User-Agent': 'Mozilla/5.0 (educational scraper)'} response = requests.get('https://example.com', headers=headers) print(response.status_code) # 200 = OK print(len(response.text)) # length of HTML string
2
Parse the HTML with BeautifulSoup
BeautifulSoup turns the raw HTML string into a tree you can navigate. Always use 'html.parser' (built-in) or 'lxml' (faster, needs pip install).
from bs4 import BeautifulSoup soup = BeautifulSoup(response.text, 'html.parser') # Now soup is a navigable tree print(soup.title.text) # page title print(soup.find('h1').text) # first h1 heading
3
Find elements with find / find_all / select
Three main ways to find elements. find() returns the first match. find_all() returns all matches as a list. select() takes a CSS selector (most powerful).
soup.find('h2') # first h2 tag soup.find('div', class_='price') # div with class="price" soup.find(id='featured') # element with id="featured" soup.find_all('a') # all links soup.find_all('li', limit=5) # first 5 list items soup.select('table tr td') # CSS: all td in table rows soup.select('.product .price') # CSS: .price inside .product
4
Extract text and attributes
From a found element, get its text content or any attribute value. Always use .strip() to remove whitespace. Use try/except for missing elements.
element = soup.find('a') element.text # all text inside the tag element.get_text() # same, with optional separator element['href'] # value of the href attribute element.get('href') # safer version (returns None if missing) element['class'] # returns a list of class names
5
Store results in a DataFrame
Collect extracted data into a list of dictionaries, then convert to a pandas DataFrame. Save to CSV for later use in your ML pipeline.
import pandas as pd records = [] for card in soup.select('.product-card'): records.append({ 'name': card.select_one('h2').text.strip(), 'price': card.select_one('.price').text.strip(), 'url': card.select_one('a').get('href', '') }) df = pd.DataFrame(records) df.to_csv('products.csv', index=False) print(df.head())

Common mistakes

✕
Scraping JavaScript-rendered pages with requests. requests only downloads the initial HTML — it doesn't execute JavaScript. If the data loads after the page (via JS), you need Selenium or Playwright instead of requests+BeautifulSoup.
✕
Not handling errors. Websites go down, return 429 (rate limited), or change structure. Always check response.status_code and wrap selectors in try/except. Use time.sleep() and retry logic.
✕
Scraping too fast. Sending hundreds of requests per second will get your IP banned and may harm the server. Always add time.sleep(1) to 3 seconds between requests. For large scrapes, use random delays.
✕
Not checking for an API first. Many sites have public APIs (Twitter, Reddit, Wikipedia, etc.) that give you cleaner, structured data faster and legally. Always search for "site_name API" before scraping.
✕
Hardcoding selectors without handling missing elements. Sites change layout. Use .get() instead of [], and check if element is not None before accessing .text. Your scraper will break on the first missing element otherwise.

Key takeaways

Summary
The basic pipeline: requests.get(url) downloads HTML → BeautifulSoup parses it → select/find locates elements → .text / ['attr'] extracts data → DataFrame stores it.
Learn CSS selectors. soup.select() is the most powerful way to find elements. Use browser DevTools (right-click → Inspect) to find the right selector for any element.
Always check robots.txt and Terms of Service. Add delays. Never scrape personal data without permission.
If content loads via JavaScript, requests won't work. Use Selenium or Playwright for dynamic pages.
Always use try/except and .get() for robust scrapers that don't crash on missing elements.
