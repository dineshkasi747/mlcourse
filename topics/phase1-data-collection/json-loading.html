JSON Loading

JSON is the most common format for API responses and semi-structured data. Unlike CSV, JSON can represent nested structures — lists inside dicts inside lists. Loading JSON into pandas requires understanding that nesting.

JSON Lines — one JSON object per line

JSON Lines (.jsonl) is a streaming-friendly format where each line is a complete JSON object. Common for log files, ML training data, and large exports. Load with pd.read_json("file.jsonl", lines=True).

When to use JSON Lines for ML: Large training datasets (millions of rows), streaming data from APIs, log files, NLP corpora. Process line-by-line with a for loop when the file is too large for RAM.

Key takeaways

Summary
Flat list of records: use pd.DataFrame(data) or pd.read_json("file.json") directly.
Nested JSON: use pd.json_normalize(data) to flatten nested objects into dot-separated column names.
JSON Lines (.jsonl): use pd.read_json("file.jsonl", lines=True) or read line by line for very large files.
Use json.loads(string) to parse a JSON string. Use json.dumps(obj) to convert Python to JSON.
After loading, always check df.info() and df.head() — nested fields often come in as object dtype.
