Simple Imputation

Real-world datasets always have missing values. Simple imputation fills those gaps using statistical summaries — mean, median, mode, or a constant — so your model can train without errors. Fast, interpretable, and often surprisingly effective.

⏱ 18 min ◆ Beginner pandas · sklearn

What is missing data, and why does it matter?

Most real datasets are incomplete. A patient skips a lab test, a sensor glitches, a survey respondent leaves a field blank. Pandas shows these as NaN. The problem: sklearn models crash on NaN — you must handle them before training.

Simple imputation replaces each NaN with a single representative value derived from the non-missing data in that column. It's the fastest fix — no new columns, no complex models, just fill and go.

Key insight: Simple imputation is a single-value fill — every NaN in a column gets the same replacement value. It's fast and always produces a complete dataset, but it shrinks variance (every imputed value is identical) and assumes the data is Missing Completely At Random (MCAR). When those assumptions fail, use KNN or iterative imputation instead.

The formulas — what SimpleImputer actually computes

1
Scan each column for non-NaN values
Collect all observed (non-missing) values X_obs in column j. The imputation statistic is computed only from observed values — missing values don't participate.
X_obs = {x_i : x_i is not NaN, i=1..n}
2
Compute the fill statistic during fit()
One statistic is stored per column. This is only computed on training data — never on test data. This prevents data leakage.
Mean: fill = sum(X_obs) / |X_obs| Median: fill = X_obs[sorted][|X_obs|//2] Most frequent: fill = argmax{count(v) : v in X_obs} Constant: fill = fill_value (user-provided)
3
Replace NaN during transform()
For every NaN in column j, substitute the stored fill statistic. Non-missing values are left unchanged.
x_i = fill_j if x_i is NaN x_i = x_i otherwise
4
Critical: fit on train, transform train+test
Always fit the imputer on training data only, then use it to transform both train and test. If you fit on the full dataset, test statistics leak into training — your model becomes overoptimistic.
imp.fit(X_train) # learn stats from train only X_train_clean = imp.transform(X_train) X_test_clean = imp.transform(X_test) # use SAME stats

Common mistakes

✕
Fitting imputer on the full dataset. imp.fit(X_all) then splitting leaks test statistics into training. Always: fit(X_train), then transform(X_train) and transform(X_test) separately, or use a Pipeline.
✕
Using mean on skewed data. Income, house prices, and durations are right-skewed — one billionaire inflates the mean dramatically. Use strategy='median' for these columns.
✕
Using mean/median on categorical columns. The mean of ["cat","dog","cat"] is meaningless. Use strategy='most_frequent' or 'constant' for string columns.
✕
Imputing without adding a missing indicator. Simple imputation destroys the signal that a value was missing. For important features, add a binary column like col_was_nan=1 before imputing so the model can use missingness as a feature.
✕
Ignoring heavily-missing columns. If 70% of a column is NaN, imputing it with the mean just creates a near-constant column with inflated confidence. Consider dropping or using a model-based imputer.

★
Key takeaways

Summary
SimpleImputer fits statistics on training data and applies them to test data — prevents leakage. Always use fit → transform, never fit_transform on the full dataset.
Use median for numeric skewed data, mean for symmetric numeric data, and most_frequent for categorical data.
Simple imputation reduces variance in the filled column — every imputed value is identical. This can hurt models sensitive to feature distributions.
Add a missing indicator column alongside imputation when missingness itself is informative (e.g., patients who skip a test may be healthier or sicker).
Always wrap imputers in a Pipeline to ensure consistent fit/transform semantics and enable grid search without leakage.
When >30–40% of a column is missing or data is MNAR, consider KNN imputation or IterativeImputer (MICE) for better results.
