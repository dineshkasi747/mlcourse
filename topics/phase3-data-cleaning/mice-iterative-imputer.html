MICE & Iterative Imputation

Multiple Imputation by Chained Equations treats imputation as a prediction problem. For each column with missing values it trains a regression model using all other columns, fills in predictions, then repeats — cycling through every column multiple times until the imputed values converge. Far superior to mean/median for MAR data.

⏱ 18 min ◆ Advanced sklearn · numpy

🔁
How MICE works — step by step

MICE (also called Fully Conditional Specification) solves the circular problem of imputing multiple columns at once by iterating. Each column's missing values are imputed using a model trained on all other columns, including other columns that were themselves just imputed. Multiple rounds of this bring all columns toward their correct conditional distribution.

0
Initialize — rough fill with mean/median
Before MICE can use other columns as predictors, it needs a starting point. It fills all missing cells with the column mean (for numeric) or mode (for categorical). This is just a rough scaffold — MICE will immediately improve on it.
X_init = SimpleImputer(strategy='mean').fit_transform(X) # All NaN cells now have a placeholder value — imputation starts
1
For each column j with missing values
Split the data into rows where column j is observed (training set) and rows where it is missing (prediction set). Use all other columns as features to train a regression/classification model on the observed rows.
for j in columns_with_missing: observed = X[X_original[:, j] is not NaN] missing = X[X_original[:, j] is NaN] model.fit(observed[:, other_cols], observed[:, j]) X[missing, j] = model.predict(missing[:, other_cols])
2
Cycle through all columns — one full pass = 1 iteration
After updating column j, immediately use the updated values when modeling column k. This means each column benefits from the latest imputed values for all other columns. One full pass through all missing columns = 1 iteration.
# After updating age using income+score, # use the updated age values when now imputing income # Each column gets smarter as the round progresses
3
Repeat for max_iter rounds — stop when converged
After enough iterations the imputed values stabilise — the change between rounds becomes negligible. sklearn's IterativeImputer defaults to 10 iterations but 3-5 is often enough for typical datasets. Track convergence by measuring the maximum absolute change per round.
for iteration in range(max_iter): prev = X.copy() for j in columns_with_missing: # ... update X[:, j] ... delta = np.max(np.abs(X - prev)) if delta < tol: break # converged
💡

Why "chained equations"? Each column's imputation model is conditioned on all other columns — forming a chain of conditional models. Running these equations in a loop (chain) is what gives MICE its name and its power.

MissingIndicator — turning NaN into a feature

Imputation fills in values, but it erases the fact that data was missing. For MNAR data — where the missingness itself is informative — you want to preserve that signal. The MissingIndicator adds a binary column for each missing feature: 1 = was missing, 0 = was present.

⚠

When NOT to add indicators: For MCAR data, adding indicators adds noise — the missingness carries no signal. Only add MissingIndicator when you have reason to suspect MNAR (missingness related to the hidden value) or MAR (missingness related to observed variables your model should know about).

Common mistakes

✕
Forgetting enable_iterative_imputer. sklearn's IterativeImputer is experimental — you must import from sklearn.experimental import enable_iterative_imputer before importing IterativeImputer, or you'll get an ImportError.
✕
Fitting on the full dataset (data leakage). MICE learns inter-feature relationships. If you fit on test data, the imputer learns the test distribution and leaks it into training. Always fit only on X_train.
✕
Using MICE on MCAR data with mean imputation as the goal. MICE is overkill for MCAR — mean/median imputation is unbiased there. MICE truly shines on MAR data where inter-feature correlations carry information about the missing values.
✕
Applying MissingIndicator to MCAR data. If missingness is completely random, the indicator column is pure noise. Adding it can hurt model performance. Only use indicators when you have evidence of MAR or MNAR.
✕
Setting max_iter=1 and wondering why MICE isn't better than mean imputation. With only 1 iteration, MICE barely improves on the mean-initialised values. Run at least 3-5 iterations for the values to converge to their conditional distributions.

★
Key takeaways

Summary
MICE imputes by treating each column as a regression target predicted by all other columns, iterating until convergence. It preserves inter-feature correlations that mean/median destroys.
Always use enable_iterative_imputer before importing IterativeImputer (sklearn experimental API).
Fit on train only — MICE learns the joint distribution of features, so fitting on test data is a serious leakage risk.
MissingIndicator should be added when missingness is MNAR — it turns "was this missing?" into a model feature, letting the algorithm exploit that signal.
3-5 iterations is usually enough. Beyond 10, improvements are marginal. Use convergence tracking (tol) to auto-stop.
MICE is significantly slower than SimpleImputer for large datasets. For very large data, consider KNNImputer as a fast middle ground between mean and MICE.
