Outlier Detection: Z-Score

A Z-score measures how many standard deviations a value sits from the mean. Values beyond ±2 or ±3 are flagged as outliers. It's the fastest, most interpretable outlier detector — one formula, one threshold, instant results.

⏱ 20 min ◆ Beginner numpy · scipy · sklearn

σ
What is a Z-score?

A Z-score answers one question: "How unusual is this value compared to the rest of the column?" It subtracts the mean and divides by the standard deviation, converting any value into a universal "deviation units" scale.

A Z-score of 0 means exactly average. A Z-score of +2 means 2 standard deviations above average — unusual but not unheard of. A Z-score of +5 means something has gone very wrong (or very right — it's extreme either way).

The 68-95-99.7 rule: For a normal distribution, ~68% of values fall within ±1σ, ~95% within ±2σ, and ~99.7% within ±3σ. So a Z-score threshold of 3 flags only the most extreme 0.3% of a normal distribution as outliers.

Modified Z-score — robust to outliers in μ and σ

The standard Z-score has a flaw: it computes μ and σ including the outliers. A few extreme values inflate both, making the Z-scores of outliers look smaller than they really are — the outliers "mask" each other. The Modified Z-score uses the median and MAD instead, which are not affected by outliers at all.

Masking problem: When you have multiple outliers, each one inflates μ and σ, which makes all their Z-scores smaller. With 5 extreme outliers, none of them may cross the threshold of 3 even though each is genuinely extreme. Use Modified Z-score (MAD-based) whenever you suspect multiple outliers or heavily contaminated data.

The formulas — standard and modified Z-score

1
Standard Z-score
Subtract the column mean and divide by the standard deviation. The result is unit-free and tells you how many standard deviations the value is from the center.
z = (x - μ) / σ where: μ = mean(X) σ = std(X) (use ddof=0 for population, ddof=1 for sample) Flag as outlier if: |z| > k (k = 2, 2.5, or 3)
2
Why Z-score fails on skewed / contaminated data
μ and σ are both non-robust statistics — a single extreme outlier shifts both significantly. This causes the "masking effect": outliers inflate σ, making their own Z-scores smaller, hiding them from detection.
Example: X = [10, 11, 12, 11, 10, 1000] μ = 175.7 σ = 393.5 z(1000) = (1000 - 175.7) / 393.5 = 2.09 ← only 2σ, not flagged at k=3! But it clearly IS an outlier.
3
Modified Z-score (Iglewicz & Hoaglin, 1993)
Replace mean with median, replace std with MAD × 1.4826 (the scale factor makes MAD consistent with σ for normal data). Now the center and spread are not influenced by outliers.
MAD = median(|X - median(X)|) M = 0.6745 * (x - median(X)) / MAD Flag as outlier if: |M| > k (k = 3.5 is the Iglewicz–Hoaglin recommendation) Note: 0.6745 = Φ⁻¹(0.75) — same scaling constant as 1/1.4826
4
Z-score for multivariate data — Mahalanobis distance
For multiple features, the standard Z-score is applied per column and misses joint outliers (e.g. a man with height=6ft and weight=60lbs — each feature is normal, but the combination is extreme). Mahalanobis distance generalises Z-score to multiple dimensions.
D² = (x - μ)ᵀ Σ⁻¹ (x - μ) # Mahalanobis distance² D ~ chi²(p) under normality, where p = n_features Flag if D > chi²_critical(p, alpha=0.001)

Common mistakes

✕
Using Z-score on skewed distributions. Z-score assumes normality — on income, house prices, or any right-skewed column, the boundaries are asymmetric and unreliable. Apply a log transform first, or use the IQR method instead for skewed data.
✕
Computing Z-score on the full dataset before splitting. The mean and std must be computed on training data only, then applied to test data. Computing on all data leaks test statistics — use a Pipeline with StandardScaler so this is handled automatically.
✕
Ignoring the masking problem with multiple outliers. Multiple outliers inflate μ and σ, masking each other. If you suspect 5+ outliers in a column, use the Modified Z-score (MAD-based) or iterative Z-score (remove and recompute) instead.
✕
Automatically dropping all detected outliers. High Z-score ≠ error. Some extremes are real (the highest-performing sales rep, the most fuel-efficient car). Always inspect flagged rows before deciding to drop, winsorize, or keep them.
✕
Using the same threshold for all columns. A Z-score of 3 flags 0.27% of a normal column, but may flag 15% of a heavy-tailed column. Tune k per column, or switch to a rank-based method like IQR for columns with unknown distributions.

★
Key takeaways

Summary
Z-score = (x − μ) / σ. It converts any value to "how many standard deviations from the mean". The threshold |z| > 3 flags the most extreme 0.27% of a normal distribution.
Only valid for roughly normal distributions. Apply log-transform first for skewed data; use IQR for heavily skewed columns; use Isolation Forest for multivariate outliers.
Use Modified Z-score (MAD-based) whenever you suspect multiple outliers — it avoids the masking problem by not letting outliers inflate the center and spread estimates.
Always fit on training data: compute μ and σ on X_train, then apply to X_test. Wrap in a Pipeline. Never compute statistics on the full dataset.
A flagged outlier is a starting point for investigation, not an automatic deletion. Check whether it's a data entry error, a valid extreme value, or a meaningful anomaly before taking action.
For multivariate outliers (unusual combinations of features), use Mahalanobis distance — the multivariate generalisation of Z-score that accounts for feature correlations.
