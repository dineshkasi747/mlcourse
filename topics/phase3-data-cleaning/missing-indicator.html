Missing Indicator

Imputation fills the gap -- but it destroys the evidence that the gap existed. A Missing Indicator adds a binary column col_was_nan=1 alongside the imputed value, letting your model learn that the fact of missingness itself is a predictive signal.

16 minBeginner-Intermediatepandas / sklearn

Why missingness is a signal, not just noise

When a patient skips a lab test, that is not random -- maybe their doctor did not order it because the result was already clear. When a customer leaves a salary field blank on a loan form, that blank predicts something about the application. The absence of a value carries information.

Imputing with the mean fills the gap -- but after imputation, the model sees only numbers and cannot distinguish rows where the value was real from rows where you made it up. A missing indicator column preserves that distinction as a clean binary signal.

Before vs after -- watch indicators appear column by column
Red = NaN  |  Orange = imputed value  |  Indigo 1 = was NaN  |  0 = was present
💡

The core idea: Imputation + Missing Indicator is always at least as good as imputation alone. If missingness has no signal, the model ignores the indicator. If it does, the model uses it. Add indicators whenever you suspect MNAR patterns.

What MissingIndicator actually does

1
Create a binary column per feature
For each feature j with any NaN values, create a boolean column: 1 where original was NaN, 0 everywhere else.
indicator_j[i] = 1 if X[i, j] is NaN indicator_j[i] = 0 otherwise Shape after: (n_samples, n_features + n_missing_features)
2
Shortcut: SimpleImputer(add_indicator=True)
The built-in add_indicator parameter imputes AND appends indicators in one transformer. Simplest and most common approach.
imp = SimpleImputer(strategy='mean', add_indicator=True) imp.fit(X_train) X_out = imp.transform(X_train) # Access which columns got indicators: imp.indicator_.features_
3
Fit on train only -- no leakage risk
MissingIndicator.fit() records which columns had NaN in training and adds indicators for those same columns in transform, ensuring consistent feature dimensionality between train and test.
mi.fit(X_train) # learn which cols had NaN mi.transform(X_test) # apply same indicator cols # train and test always have the same number of features

Common mistakes

✕
Adding indicators for MCAR features. If missingness is random, the indicator column is pure noise. Check whether missingness correlates with the target before adding.
✕
Fitting MissingIndicator on test data. Always fit on training data only. Using test data creates train/test shape mismatches and leakage.
✕
Adding an indicator but forgetting to also impute. Use SimpleImputer(add_indicator=True) to do both atomically.
✕
Dropping the indicator after EDA shows low correlation. Low marginal correlation does not mean useless in combination with others. Let the model decide via regularisation or feature importance.
✕
Not naming indicator columns explicitly. Use imp.indicator_.features_ to get the indices and name them col_name + "_was_nan" for readable feature importance plots.

★
Key takeaways

Summary
A missing indicator adds a binary column that is 1 where the original feature was NaN -- preserving the signal that missingness itself is informative.
Use it whenever data is MNAR -- when the reason something is missing correlates with the target: medical tests not ordered, financial info not disclosed, user has no history.
The simplest implementation: SimpleImputer(add_indicator=True) -- imputes and adds indicators in one transformer, fit on training data only.
Adding an indicator to MCAR features is harmless but wasteful -- regularised models zero it out, tree models ignore it. Add liberally and let the model decide.
Always wrap in a Pipeline. The imputer must be fit on training data only -- Pipeline ensures consistent fit/transform semantics automatically.
