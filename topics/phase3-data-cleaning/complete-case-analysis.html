When CCA is appropriate — decision guide

✓
Use CCA when
Missingness is MCAR and <5% of rows are affected
When data is missing completely at random and very little of it, dropping rows loses minimal information and introduces no bias. This is the only situation where CCA is unambiguously safe.
missing_pct = df.isnull().any(axis=1).mean() if missing_pct < 0.05: # less than 5% rows affected df_clean = df.dropna() # safe to use CCA
✓
Use CCA when
You need a quick baseline model
For a first quick model to establish a baseline, CCA is fast and simple. You can always come back with better imputation later. Document that you used CCA as a baseline.
df_baseline = df.dropna() # fast baseline model.fit(df_baseline.drop('target', axis=1), df_baseline['target']) # Later: replace with proper imputation pipeline
x
Avoid CCA when
More than 5-10% of rows would be dropped
At 5% missing per column with 10 columns you lose 40% of data. At 20 columns you lose 64%. The loss compounds multiplicatively. Use imputation instead.
rows_lost = 1 - (1 - p_miss_per_col) ** n_cols # Example: p=0.05, n=10 → 40% rows lost # If rows_lost > 0.05: use imputation instead
x
Avoid CCA when
Missing is MAR or MNAR
If younger people are less likely to report income (MAR), dropping them gives you a sample biased toward older people. Your model learns from the wrong distribution. The bias is systematic and silent.
# Detect MAR: correlate missingness indicator with other columns miss_ind = df['income'].isnull().astype(int) # If corr(miss_ind, age) is large → MAR → do NOT use CCA

Common mistakes

x
Using CCA without checking how many rows are lost. Always print df.shape before and after dropna(). If you lose more than 5%, switch to imputation.
x
Using CCA on MAR data. The rows you drop are a non-random subset. Your model trains on a biased sample and will fail on the full production distribution.
x
Forgetting that df.dropna() drops the entire row for one missing value. A row with 19 perfect values and 1 missing is dropped entirely. Consider dropping only columns with high missingness first.
x
Applying CCA after train-test split on the test set. Never drop rows from the test set — you need to evaluate on all rows. Only clean the training data.

*
Key takeaways

Summary
CCA (dropna) is safe ONLY under MCAR with <5% rows affected. All other cases need imputation.
Row loss compounds: with p=10% per column and k=10 columns, you lose 65% of rows.
Always compare distribution before and after dropna() to detect bias.
CCA is useful for quick baselines, but always replace with a proper imputation pipeline.
Never drop rows from the test set — only clean training data.
