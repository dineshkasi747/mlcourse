Types of Missing Values

Not all missing values are equal. Why a value is missing determines what you can safely do about it. There are three distinct mechanisms — MCAR, MAR, and MNAR — and confusing them leads to biased models and wrong conclusions.

⏱ 18 min ◆ Beginner–Intermediate pandas · missingno

?
Three mechanisms — MCAR, MAR, MNAR

Before choosing a strategy to handle missing data, you must understand why the data is missing. The mechanism determines whether simple imputation is safe or whether you need a more careful approach.

MCAR
Missing Completely At Random
The probability of being missing has nothing to do with any variable — observed or unobserved.
Safest — any imputation OK
MAR
Missing At Random
Probability of being missing depends on other observed variables — but not on the missing value itself.
Manageable with care
MNAR
Missing Not At Random
Probability of being missing depends on the missing value itself — the data that is missing is the data you most need.
Hardest — needs domain knowledge
MCAR — random dropout, no pattern
MCAR example: A survey respondent accidentally skips a question. A sensor randomly glitches. A blood pressure cuff randomly fails. There is no pattern — the missingness is truly random. Safe to use simple imputation or complete-case analysis without introducing bias.

How to detect the type — statistical tests

You can never be 100% certain about the missing data mechanism without domain knowledge. But statistical tests can give strong evidence. The most useful: compare the distribution of observed variables between rows where a column is missing vs present.

Detection checklist — run through these steps
Step 1 — How much is missing?
Run df.isnull().sum(). Less than 5% can often be dropped. 5-40% needs imputation. Over 40% — consider if the column is useful at all.
Step 2 — Is there a pattern?
Compare means of other columns between missing and non-missing groups. If they differ significantly — MAR. Use t-test or Little's MCAR test.
Step 3 — Domain knowledge
Ask: Why would this value be missing? If the answer is "because of its own value" (e.g., rich people hide income) — MNAR. This cannot be detected statistically alone.
Step 4 — Visualise
Plot histograms of other features split by whether target feature is missing. Use msno.matrix(df) or msno.heatmap(df) from the missingno library.

Which strategy to use — decision table

MechanismSafe strategiesRisky strategiesBias risk
MCAR Delete rows · Mean/median imputation · KNN imputation · Model-based imputation None — all work if missingness is truly random Low — reduces sample size but no bias
MAR Multiple imputation · KNN imputation · Model-based (use observed predictors) Complete-case deletion (loses information) · Simple mean (ignores predictors) Medium — bias if predictors not used
MNAR Missing indicator feature · Domain-specific model · Sensitivity analysis Any single imputation strategy (all introduce bias) High — no statistical fix; needs domain knowledge
⚠

The MNAR problem: When data is MNAR, there is no purely statistical solution. The missing values are systematically different from observed values in ways you cannot measure. The best approach: create a binary "was_missing" indicator column, use domain knowledge to reason about the bias, and report uncertainty in your model's conclusions.

Common mistakes

✕
Assuming all missing data is MCAR. Most real-world missing data is MAR or MNAR. Blindly using mean imputation or dropping rows without checking for patterns leads to biased models that fail in production.
✕
Dropping all rows with any missing value. Complete-case analysis is only unbiased for MCAR. For MAR/MNAR it removes a non-random subset of your data, making your training set unrepresentative.
✕
Imputing before the train/test split. If you compute the mean on the whole dataset before splitting, test data "leaks" information into training. Always fit the imputer on training data only, then transform both train and test.
✕
Ignoring the missingness signal itself. For MNAR data, the fact that a value is missing IS information. Create a binary feature_was_missing column before imputing — this often improves model performance significantly.

★
Key takeaways

Summary
MCAR = truly random dropout. Any imputation strategy works. Safe to delete rows too.
MAR = missing depends on other observed columns. Use model-based or KNN imputation that exploits those relationships.
MNAR = missing depends on the value itself. No statistical fix — use domain knowledge + missing indicator feature.
Always visualise the missingness pattern before choosing a strategy. Random scatter = MCAR. Systematic bands = MAR or MNAR.
Always fit imputers on training data only — never on the full dataset before splitting.
