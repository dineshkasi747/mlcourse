Filter Methods

Score each feature independently of the model using statistical tests. Fast, scalable, and model-agnostic. The first line of defence against irrelevant and redundant features.

22 min Intermediate sklearn · scipy

Variance Threshold
Univariate
Removes features with variance below a threshold. Catches constant or near-constant columns that have no predictive signal.
Correlation with Target
Pearson / Spearman
Ranks features by absolute Pearson or Spearman correlation with the target. Fast but only captures linear relationships.
Chi-Squared Test
Categorical target
Tests statistical dependence between feature and target. Only valid for non-negative features and classification problems.
ANOVA F-test
Classification
Tests if feature means differ significantly across classes. Assumes normal distribution within classes. sklearn's f_classif.
Mutual Information
Non-linear
Measures dependency regardless of relationship shape. Captures non-linear associations that Pearson misses. More expensive to compute.
VIF (Redundancy)
Anti-collinearity
Variance Inflation Factor detects multicollinearity. Drop features with VIF > 10. Complements relevance-based methods.

Common mistakes

x
Using only Pearson correlation for non-linear features. Pearson scores a sinusoidal feature as near 0. Always pair it with mutual information to catch non-linear relationships.
x
Applying filter methods to the full dataset before train-test split. The scores are computed from data the model hasn't seen yet — this is leakage. Fit filter scores only on training data.
x
Using chi-squared on negative or continuous features. sklearn's chi2 requires non-negative values. Standardise and min-max scale first, or use f_classif/mutual_info_classif instead.
x
Relying on a single filter method. Each method has blind spots. Use 2-3 methods and keep features that rank well on at least two — this is ensemble filter scoring and is much more robust.
x
Ignoring feature interactions. Filter methods score each feature independently. A feature with a low individual score might be critical in combination with another feature. Wrapper or embedded methods handle this better.

*
Key takeaways

Summary
Filter methods score features independently of any model — fast and scalable to thousands of features.
Always start with VarianceThreshold to remove constant/near-constant features before anything else.
Pearson correlation: fast, linear only. Mutual information: slower, captures non-linear. Use both.
Use f_classif for classification, f_regression for regression, chi2 only for non-negative features.
Ensemble scoring: keep features that rank well across 2+ methods — much more robust than a single method.
Always fit filter scores on training data only — computing them on the full dataset is data leakage.
