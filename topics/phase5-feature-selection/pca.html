Principal Component Analysis

PCA finds the directions of maximum variance in your data and projects everything onto those directions. 100 correlated features become 5 independent ones that still capture 95% of the information. Every step has live math and visuals below.

⏱ 25 min ◆ Intermediate numpy · sklearn

➡
The core idea — finding the best "shadow"

Imagine shining a flashlight on a 3D object and watching its shadow on the wall. PCA finds the angle of the flashlight that casts the most informative shadow — the one where the shadow is most spread out (highest variance). That spread means you are preserving the most differences between points.

Drag the points below or rotate the projection axis to see how variance changes. The red arrow is PC1 — the direction that maximises the spread of the projected dots.

Key insight: The angle that maximises projected variance is exactly the first eigenvector of the covariance matrix. PCA is just finding these eigenvectors — one per dimension — each orthogonal to all the previous ones.

Standardize — center and scale every feature
⚠ PCA is scale-sensitive sklearn: StandardScaler

The problem: if feature A is in millimetres (0–2000) and feature B is in percentages (0–1), PCA ignores B — not because it is unimportant, but because its numbers are smaller. PCA sees scale as importance.

Two operations, one goal
① Center: subtract column mean → data sits at origin
② Scale: divide by column std → every feature has variance = 1
X_std = (X - X.mean(axis=0)) / X.std(axis=0) Result: mean = 0.0, std = 1.0 for every column

2
Covariance matrix — measure how features move together
Shape: (n_features, n_features) Symmetric matrix

What each cell means: C[i][j] tells you how much feature i and feature j vary together. Positive = they rise and fall together. Negative = they move in opposite directions. Zero = independent.

Diagonal
C[i][i] = variance of feature i. How spread out each feature is.
Off-diagonal
C[i][j] = covariance. High value = features are correlated → PCA can compress them.
C = X_std.T @ X_std / (n - 1) trace(C) = sum of variances = total variance High off-diag = high correlation = PCA helps!

3
Eigendecomposition — find the principal directions
Core of PCA numpy.linalg.eigh(C)

What is an eigenvector? A special direction that the covariance matrix only stretches, never rotates. The amount of stretch is the eigenvalue λ.

Eigenvalue λ → how much variance this direction captures. Bigger = more important.
Eigenvector v → the actual direction in feature space. This becomes a Principal Component.
Sort by λ descending → PC1 has the most variance, PC2 next, and so on.
C @ v = λ * v λ_1 ≥ λ_2 ≥ ... ≥ λ_n (sort descending) v_1, v_2, ... are orthogonal unit vectors

Project — transform data into the new PC space
Dimensionality reduction happens here pca.transform(X)

One dot product per component: for each data point, compute how much it aligns with each eigenvector. That alignment score is the PC coordinate.

Shape transformation
Input:  (n_samples, n_features)
V_k:    (n_features, k)    k chosen components
Output: (n_samples, k)   ← compressed!
Z = X_std @ V_k # project X_rec = Z @ V_k.T # reconstruct (optional) Each Z[:, i] is PC(i+1) scores for all samples

⚠

The information loss trade-off: With k=1 component on correlated 2D data you might keep 85% of variance but lose 15%. For most ML tasks, 95% variance explained is a good target. Always check that your downstream model accuracy doesn't drop significantly after PCA.

Common mistakes

✕
Not standardizing before PCA. PCA finds directions of maximum variance. If feature A has values 0–1 and feature B has values 0–1000, PCA will almost entirely focus on B just because of its scale. Always use StandardScaler before PCA.
✕
Fitting PCA on test data. The same leakage rule applies. Fit PCA on training data only, then transform both train and test. The principal components should be computed from training data alone.
✕
Using PCA when features are not correlated. PCA benefits from inter-feature correlation — it "consolidates" redundant information. If all features are already independent, PCA won't help and may hurt. Check your correlation matrix first.
✕
Applying PCA to categorical features. PCA assumes continuous numeric data with meaningful distances. One-hot encoded features and ordinal categories don't satisfy this. Use MCA (Multiple Correspondence Analysis) for categorical data instead.
✕
Choosing n_components without checking downstream accuracy. 95% explained variance sounds safe but might drop accuracy on a specific task. Always validate with cross-validation on the actual task, not just the explained variance metric.

★
Key takeaways

Summary
PCA finds the eigenvectors of the covariance matrix — these are the directions of maximum variance. Project data onto the top-k eigenvectors to compress from n to k dimensions.
Always standardize first. PCA is not scale-invariant. Features with large ranges dominate. Use StandardScaler before PCA in a Pipeline.
Choose n_components by targeting 95% explained variance as a starting point, then validate on your downstream task. Use the scree plot to spot the elbow.
The biplot shows which original features contribute most to each PC (loading vectors). Long arrows = high importance. Parallel arrows = correlated features.
PCA is a linear method. For nonlinear structure, use Kernel PCA, UMAP, or t-SNE instead.
Fit on train only. Wrap StandardScaler + PCA in a Pipeline to make this automatic and leak-proof.
