Wrapper Methods

Train the model repeatedly on different feature subsets and keep the subset that scores best. Slower than filter methods, but actually accounts for how features interact inside the model.

20 min Intermediate sklearn

?
What is a wrapper method?

The idea is simple: use the model itself to judge which features are useful. You train the model with different subsets of features and keep the subset with the best cross-validation score.

i

Filter vs Wrapper in one sentence: Filter methods score each feature individually using statistics — no model involved. Wrapper methods score feature subsets by actually training the model. This means wrappers can find that "feature A alone is useless, but A + B together are powerful" — something filters completely miss.

There are three main wrapper strategies. They only differ in how they search the space of possible subsets:

Forward Selection
Start with no features. Each step: try adding every remaining feature, keep the one that improves the model the most. Stop when nothing improves it.
{} → {A} → {A,C} → {A,C,F} ✓
Backward Elimination
Start with all features. Each step: remove the one feature whose removal hurts performance the least. Stop when any removal hurts.
{A,B,C,D} → {A,C,D} → {A,C} ✓
RFE (Recursive)
Like backward elimination, but uses the model's own feature weights to decide which to cut next. Cheaper and usually better.
sklearn: RFE(estimator, n_features_to_select=5)

Computational cost — how many models get trained?

The main downside of wrapper methods is cost. With 5-fold CV it multiplies fast. This is why you should filter first, then wrap.

MethodModels trained (p features, k=5 CV)p=20 exampleWhen to use
Forward / Backward k × p(p+1)/2 5 × 210 = 1,050 p < 30, fast model
RFE k × p (one fit per elimination) 5 × 20 = 100 Default choice — use this
RFECV k × p (CV also finds optimal k) 5 × 20 = 100 Best all-around: auto picks feature count
Exhaustive k × 2p 5 × 1M = 5M+ Only for p < 15
!

Practical rule: Use filter methods first to cut to p < 30 features, then apply RFECV. This keeps total model fits low while still benefiting from wrapper-quality selection.

Common mistakes

x
Running wrapper on the full dataset, then evaluating. The selection step itself must be inside cross-validation. Use Pipeline so RFE only sees training fold data.
x
Applying wrapper methods to 100+ features directly. Filter first. Cut to <30 features with VarianceThreshold + mutual information, then wrap.
x
Wrapping with a slow model. Use Logistic Regression or Decision Tree as the base estimator for selection. Save your expensive model for after features are chosen.
x
Forgetting to scale before RFE with linear models. RFE uses coefficient magnitudes to rank features. If features are on different scales, Lasso/LogReg coefficients are unfair. Always StandardScale first.

*
Key takeaways

Summary
Wrapper methods evaluate feature subsets by training the model — they catch interactions that filter methods miss.
Use RFECV as the default. It is cheap (k×p fits), automatically finds the optimal feature count, and gives you a learning curve to inspect.
Always filter first to <30 features, then wrap. This keeps cost low.
Wrap with a fast model (LogReg, Decision Tree). Train your actual model on the selected features separately.
Put the wrapper inside a Pipeline so feature selection never sees test fold data.
