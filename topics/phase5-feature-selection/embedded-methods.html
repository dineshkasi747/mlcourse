Embedded Methods

Feature selection happens during model training — not before or after. Lasso drives irrelevant coefficients to exactly zero. Tree models compute importance at every split. The best of both worlds: selection + modelling in one step.

25 min Intermediate sklearn · numpy

Why L1 creates sparsity but L2 doesn't: L1 penalty has a "corner" at zero in every dimension — the gradient of |w| is discontinuous at 0. This makes the optimum land exactly at zero for unimportant features. L2 (Ridge) has a smooth penalty that shrinks towards zero but never reaches it.

RF
Tree-based importance — three methods compared

Tree models provide three distinct importance measures. Switch between them — they tell very different stories. Permutation importance is the most reliable for model selection.

Impurity-based importance (MDI)
Total reduction in node impurity weighted by samples — fast but biased toward high-cardinality features
Dataset:
Impurity (MDI)
Fast to compute. Biased toward high-cardinality & continuous features. Can overestimate importance of noisy features. Available from model.feature_importances_.
Permutation
Shuffles each feature and measures performance drop. Unbiased, works on any model. Slower. Correlated features share importance. Use on validation set for honest estimate.
SHAP values
Shapley values from game theory. Shows direction + magnitude per sample. Most informative. Gold standard for interpretability. Handles interactions & correlated features correctly.

4
How embedded methods work — step by step

1
Lasso / ElasticNet
Add L1 penalty to loss — irrelevant coefficients go to exactly zero
The L1 penalty term creates sparsity. The optimal λ must be found via cross-validation. Use LassoCV to search automatically.
from sklearn.linear_model import LassoCV model = LassoCV(cv=5, random_state=42) model.fit(X_train, y_train) selected = X.columns[model.coef_ != 0] print(f"Selected {len(selected)} features")
2
Random Forest / XGBoost
Train tree ensemble — read feature_importances_ from the model
Each split in each tree reduces impurity. The total weighted reduction per feature = MDI importance. Use permutation importance for a more reliable estimate on the validation set.
from sklearn.ensemble import RandomForestClassifier from sklearn.inspection import permutation_importance rf = RandomForestClassifier(n_estimators=100).fit(X_train, y_train) # MDI (fast, biased): mdi = rf.feature_importances_ # Permutation (slower, unbiased): pi = permutation_importance(rf, X_val, y_val, n_repeats=10)
3
SelectFromModel
Use sklearn's SelectFromModel to automate the cut
SelectFromModel wraps any model that exposes coef_ or feature_importances_ and returns only features above the threshold. Works inside a Pipeline to prevent leakage.
from sklearn.feature_selection import SelectFromModel from sklearn.pipeline import Pipeline pipe = Pipeline([ ('select', SelectFromModel(RandomForestClassifier(100))), ('clf', LogisticRegression()) ]) pipe.fit(X_train, y_train)
4
ElasticNet — best of both
Mix L1 + L2 for grouped sparsity
ElasticNet combines L1 sparsity with L2 grouping. When features are correlated, Lasso picks one arbitrarily. ElasticNet keeps the whole group — better when correlated features are all relevant.
from sklearn.linear_model import ElasticNetCV # l1_ratio=1.0: pure Lasso, l1_ratio=0.0: pure Ridge en = ElasticNetCV(l1_ratio=[0.1,0.5,0.7,0.9,1.0], cv=5) en.fit(X_train, y_train) print(f"Best l1_ratio: {en.l1_ratio_}")

Common mistakes

x
Using MDI importance without permutation importance. MDI is biased toward high-cardinality and continuous features. Always cross-check with permutation importance on a validation set.
x
Not scaling before Lasso. Lasso penalises coefficients — if features are on different scales, the penalty is applied unequally. Always StandardScale before applying Lasso/ElasticNet.
x
Using a single λ value without cross-validation. LassoCV automatically searches the regularisation path. A single manually chosen alpha is almost certainly suboptimal.
x
Trusting Lasso when features are correlated. Lasso picks one feature from a correlated group arbitrarily. If both are important, use ElasticNet which tends to select the whole group.
x
Computing permutation importance on the training set. Permutation importance on training data inflates importance of overfit features. Always compute it on a held-out validation set.

*
Key takeaways

Summary
Embedded methods do selection during training — no separate step needed. Most efficient approach.
Lasso (L1) drives irrelevant coefficients to exactly zero. Always use LassoCV to find optimal λ.
MDI (impurity importance) is fast but biased. Always verify with permutation importance on validation data.
ElasticNet = L1 + L2. Better than pure Lasso when correlated features are all genuinely useful.
Use SelectFromModel inside a Pipeline to apply embedded selection without data leakage.
SHAP values are the gold standard for importance — they handle interactions and correlated features correctly.
