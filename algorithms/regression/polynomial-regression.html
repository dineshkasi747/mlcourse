Polynomial Regression

Fit curves instead of lines by adding powers of x as new features. Degree 1 = straight line. Degree 2 = parabola. Degree 10 = wiggly mess. The perfect playground to see underfitting and overfitting happen live.

⏱ 18 min ◆ Beginner numpy · sklearn

〜
Intuition — turning a line into a curve

Imagine you're trying to draw the best line through data that clearly bends like a hill. A straight line will always be wrong — it can't bend. The trick? Add new features: x², x³, x⁴ ... and let linear regression do the rest. The model stays "linear" in its parameters (β₀, β₁, β₂...) — just the features become curved.

Think of it like this: instead of asking "what's the best y = ax + b line?", you ask "what's the best y = ax² + bx + c parabola?" — same OLS math, just richer features.

🧠

Mental model for beginners: A degree-1 polynomial is a ruler — perfectly straight. Degree-2 is a flexible ruler that bends once. Degree-10 is a wet noodle — bends wildly to touch every single point. You want the ruler that's just flexible enough to follow the real pattern without memorising the noise.

Bias-Variance Tradeoff — the central lesson

Every model makes two kinds of error: bias (too simple — misses the real pattern) and variance (too complex — fits noise instead of pattern). Polynomial degree directly controls this tradeoff.

Underfitting
d=1
High Bias · Low Variance
Line can't follow a curved truth. Wrong on training AND test data. The model is too simple to learn the real pattern.
Just Right ✓
d=3
Low Bias · Low Variance
Captures the real curve without chasing noise. Good on both train and test. This is the goal.
Overfitting
d=12
Low Bias · High Variance
Memorises training noise perfectly. Train R²≈1 but test R² collapses. Useless on new data.

Rule of thumb: Use cross-validation to pick degree. Start with degree 2–4. If train R² ≫ test R², you're overfitting — lower degree or increase α. If both are low, underfitting — try higher degree or a non-linear model entirely.

How it works — feature expansion then OLS

1
Transform: create polynomial features
For each original feature x, create new features x², x³, ..., x^d. This is called feature expansion. The model is still linear in these new features — that's the trick.
Original: X = [x₁, x₂, ..., xₙ] Degree-3 expansion: X_poly = [1, x, x², x³] ↑ ↑ ↑ ↑ bias linear quad cubic
2
Fit: apply OLS to the expanded features
Exactly the same Normal Equations as MLR, just with more columns. The loss is still sum of squared residuals.
ŷ = β₀ + β₁x + β₂x² + β₃x³ = X_poly · β β̂ = (X_poly^T X_poly)^{-1} X_poly^T y
3
Why high degree causes overfitting
With degree d and n training points, you have d+1 parameters. When d+1 ≈ n, the model has enough freedom to pass through every single training point — memorising noise instead of pattern.
If n = d + 1: training error = 0 (perfect fit) But test error → ∞ (completely useless) Parameters = d + 1 (intercept + d coefficients) Rule: need n >> d+1 for generalisation
4
Fix: Ridge regularisation on polynomial features
Add an L2 penalty to shrink large coefficients. This prevents the model from creating wild oscillations even with high degree. The trade-off: slightly higher train error, much better test error.
Regularised loss: L(β) = ||y - X_poly β||² + α ||β||² Solution: β̂ = (X_poly^T X_poly + αI)^{-1} X_poly^T y ↑ adds α to diagonal → prevents singularity

Common mistakes

✕
Not scaling features before polynomial expansion. x=1000 becomes x²=1,000,000. The huge range causes numerical instability. Always use StandardScaler before PolynomialFeatures.
✕
Choosing degree by looking at training R². Training R² always goes up with degree. Always use cross-validation or a held-out test set to pick degree.
✕
Using degree > 5 without regularisation. High-degree polynomials with no penalty always overfit on real data. Use Ridge (alpha > 0) when using degree > 4.
✕
Trusting polynomial fits at the edges of your data. Polynomial curves go to ±∞ outside the training range. Never extrapolate beyond your data range with polynomial models.
✕
Forgetting that polynomial features explode with multiple features. 2 features at degree 3 = 9 features. 5 features at degree 4 = 70 features. Use interact_only=True or consider tree-based models instead.

★
Key takeaways

Summary
Polynomial regression is still linear regression — just with engineered features (x, x², x³...). Same math, same OLS solver.
Degree is the key hyperparameter. Low degree = underfitting (high bias). High degree = overfitting (high variance). Pick with cross-validation.
Always plot train R² vs test R² across degrees. The gap between them is the overfitting signal.
Always scale before expanding features. StandardScaler → PolynomialFeatures → LinearRegression (or Ridge).
Ridge regularisation lets you use higher degrees safely. RidgeCV auto-tunes α.
For multiple features, polynomial expansion causes a feature explosion. Consider tree-based models (Random Forest, XGBoost) as alternatives.
