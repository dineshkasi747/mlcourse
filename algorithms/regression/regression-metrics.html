Regression Metrics

How do you know if your regression model is good? You need a number that summarises prediction quality. Different metrics punish errors differently — choosing the right one matters as much as choosing the right model.

MAE — Mean Absolute Error

MAE measures the average absolute difference between predictions and actual values. Every error is treated equally — a miss of 10 is exactly twice as bad as a miss of 5. It is in the same units as your target variable, so easy to interpret.

Outlier sensitivity — why MAE vs MSE matters

The most important practical difference: MSE/RMSE punish large errors much harder than MAE. Squaring a big error (e.g. 100) gives 10,000 — it dominates the metric. MAE treats every error proportionally. Drag the outlier below and watch how each metric reacts.

⚠

Rule of thumb: If outliers are real and important (e.g. fraud, safety), use RMSE — it will punish those big misses hard. If outliers are noise or not meaningful (e.g. sensor glitches), use MAE — it ignores outlier severity.

R²
R² explained — what does "coefficient of determination" mean?

R² answers: "How much better is my model than just always predicting the mean?" R² = 1 means perfect predictions. R² = 0 means your model is no better than the mean baseline. R² can be negative — meaning your model is worse than the mean!

💡

Adjusted R² penalises adding useless features. Regular R² always increases when you add more features — even random noise. Adjusted R² decreases if a new feature doesn't actually help. Always use Adjusted R² when comparing models with different numbers of features.

Common mistakes

✕
Using R² alone to judge a model. R² = 0.95 sounds great, but if your model always predicts the mean shifted by a constant, R² can still be high. Always also look at a residual plot and MAE/RMSE.
✕
Using MAPE when y can be zero or near-zero. MAPE divides by the actual value — if y = 0, you get a division by zero. If y = 0.001, one small prediction error becomes 10,000%. Use MAE or RMSE instead.
✕
Comparing MSE across different datasets. MSE depends on the scale of y. A model with MSE = 100 on house prices (in thousands) might be excellent, while MSE = 100 on test scores (0-100) is terrible. Use R² or normalised metrics to compare across datasets.
✕
Ignoring residual plots. A single number metric can look fine while hiding systematic patterns (heteroscedasticity, non-linearity). Always plot actual vs predicted and residuals vs predicted.

★
Key takeaways

Summary
MAE = average absolute error. Same unit as y. Not sensitive to outliers. Easy to explain to stakeholders.
MSE = average squared error. Heavily penalises large errors. Useful as a loss function during training.
RMSE = square root of MSE. Same unit as y. Best for evaluation when large errors are costly.
R² = how much variance is explained (0 to 1). Use for model comparison. Use Adjusted R² when comparing different numbers of features.
MAPE = percentage error. Scale-independent. Never use when y can be zero.
Always pair any metric with a residual plot — metrics alone can hide serious model problems.
