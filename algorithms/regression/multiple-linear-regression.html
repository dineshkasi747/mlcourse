📐
Intuition — fitting a hyperplane

Simple linear regression fits a line through 2D data. Multiple linear regression fits a hyperplane through n-dimensional data. With 2 features you get a flat plane through 3D space; with p features you get a p-dimensional surface you can't visualise but the math is identical.

Each coefficient βj says: "Holding all other features constant, a 1-unit increase in feature j changes the prediction by βj." This partial effect interpretation is what makes MLR so powerful for understanding data.

Key insight: Coefficients are partial effects. If x₁ and x₂ are correlated, changing x₁ changes x₂ too — but β₁ tells you the effect of x₁ alone, as if x₂ were held fixed. This is why multicollinearity causes instability: correlated features fight each other for credit.

Multicollinearity — when features fight each other

When two features are highly correlated, the model struggles to assign credit properly. The coefficients become unstable — small data changes cause huge coefficient swings. The VIF (Variance Inflation Factor) detects this: VIF > 10 is a serious problem.

⚠

Signs of multicollinearity: R² is high but individual coefficients aren't significant, coefficient signs flip unexpectedly, small data changes cause large coefficient changes, VIF > 10 for any feature. Fix: remove one correlated feature, use PCA, or apply Ridge regularisation.

✓
OLS Assumptions — when MLR works perfectly

OLS (Ordinary Least Squares) has the best possible estimator properties (BLUE — Best Linear Unbiased Estimator) only when all five assumptions hold:

📏
Linearity
y is a linear function of features. Test with scatter plots, residual vs fitted plots.
🎲
Independence
Observations are independent. Violated in time series (autocorrelation). Test: Durbin-Watson.
⚖️
Homoscedasticity
Residual variance is constant across all fitted values. Test with residual vs fitted plot.
🔔
Normality of Residuals
Residuals follow a normal distribution. Test with Q-Q plot or Shapiro-Wilk test.
🔗
No Multicollinearity
Features are not highly correlated with each other. Test: VIF. Fix: Ridge regression.

The OLS normal equations — step by step

1
Model definition
MLR assumes y is a linear combination of p features plus Gaussian noise ε ~ N(0, σ²).
ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₚxₚ Matrix form: ŷ = Xβ where X ∈ ℝⁿˣ⁽ᵖ⁺¹⁾, β ∈ ℝᵖ⁺¹ (first col of X = ones for intercept)
2
Loss function — Sum of Squared Residuals
Minimise the total squared error between predictions and actual values.
L(β) = ||y - Xβ||² = (y - Xβ)ᵀ(y - Xβ) = Σᵢ(yᵢ - β₀ - β₁x_{i1} - ... - βₚx_{ip})²
3
Take gradient, set to zero
Differentiate L with respect to β and set to zero — this gives the Normal Equations.
∂L/∂β = -2Xᵀ(y - Xβ) = 0 → Xᵀy = XᵀXβ ← Normal equations
4
Closed-form solution (OLS estimator)
If XᵀX is invertible (no perfect multicollinearity), we get the unique global minimum analytically.
β̂ = (XᵀX)⁻¹ Xᵀy Complexity: O(p²n + p³) — cubic in features! For large p: use gradient descent instead.
5
Why OLS is BLUE (Gauss-Markov theorem)
Under the 5 OLS assumptions, β̂ is the Best Linear Unbiased Estimator — it has the minimum variance among all linear unbiased estimators.
Var(β̂) = σ²(XᵀX)⁻¹ Std error of β̂ⱼ = σ √[(XᵀX)⁻¹]ⱼⱼ t-statistic: t = β̂ⱼ / SE(β̂ⱼ) → p-value tells significance

Common mistakes

✕
Interpreting coefficients as causal effects. β₁ measures association, not causation. Omitted variable bias — an unobserved feature correlated with both x₁ and y — will contaminate all coefficients.
✕
Ignoring multicollinearity. R² looks great but SE(β̂) explodes. Always compute VIF. Remove one of a correlated pair or use Ridge regression.
✕
Not scaling before comparing coefficients. If x₁ is in km and x₂ is in mm, β₁ will be tiny and β₂ huge even if x₁ matters more. Standardise first to compare magnitudes.
✕
Adding too many features without regularisation. With n=50 samples and p=40 features, OLS will overfit wildly (train R²=1, test R²<0 possible). Use adjusted R² or cross-validation.
✕
Forgetting to check residual plots. High R² doesn't mean good model. A curved residual pattern means linearity is violated. Residual fan shape means heteroscedasticity. Always plot residuals.

★
Key takeaways

Summary
MLR fits a hyperplane β̂ = (XᵀX)⁻¹Xᵀy that minimises the sum of squared residuals.
Each coefficient is a partial effect — the change in y per unit of xⱼ with all other features held constant.
Adjusted R² penalises adding useless features. Always report adjusted R², never raw R².
VIF > 10 signals severe multicollinearity. Fix: remove one correlated feature or switch to Ridge regression.
Always check residual plots: vs fitted (linearity + homoscedasticity), Q-Q (normality), and scale-location (heteroscedasticity).
MLR is the foundation for Ridge, Lasso, ElasticNet — they all modify the same OLS objective with a regularisation penalty.
