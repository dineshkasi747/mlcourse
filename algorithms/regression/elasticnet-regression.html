Ridge + Lasso combined

ElasticNet Regression

Ridge keeps all features. Lasso zeros some out but struggles when features are correlated. ElasticNet combines both — you get sparsity from Lasso AND stability from Ridge, controlled by a single dial called l1_ratio.

⌛ 18 min ◆ Intermediate numpy · sklearn

◆
What is ElasticNet? — the best of both worlds

You have learned Ridge and Lasso. Here is the problem each one has:

  • Ridge — keeps ALL features (no zeros). Bad when you want to know which features matter.
  • Lasso — zeros features out. But when two features are highly correlated, it randomly picks one and drops the other. Unstable.

ElasticNet fixes both problems. It mixes Ridge and Lasso using a parameter called l1_ratio. Setting l1_ratio=0 gives you pure Ridge. Setting l1_ratio=1 gives you pure Lasso. Anything in between is ElasticNet.

💡

Simple analogy: Imagine mixing hot and cold water with two taps. Ridge is the cold tap (stable, no zeros). Lasso is the hot tap (sparse, zeros things out). ElasticNet's l1_ratio is the dial that mixes them. You choose the temperature that works best for your data.

Ridge (L2)
l1_ratio = 0
Shrinks all coefficients toward zero. No feature is fully removed. Great when all features matter a little.
ElasticNet
l1_ratio = 0.5
Mix of both. Some features get zeroed (Lasso part). Correlated features are handled stably (Ridge part).
Lasso (L1)
l1_ratio = 1
Zeros out weak features completely. Unstable with correlated features — randomly picks one from a correlated group.

The formula — one line that says it all

1
ElasticNet combines both penalties
Start with the plain OLS loss (sum of squared errors). Then add both the Ridge penalty (sum of squared coefficients) and the Lasso penalty (sum of absolute coefficients). The l1_ratio controls how much weight each gets.
ElasticNet loss = fitting error + alpha * l1_ratio * |beta| (Lasso part) + alpha * (1 - l1_ratio) * beta^2 (Ridge part) When l1_ratio = 1: only Lasso part -> exact zeros possible When l1_ratio = 0: only Ridge part -> no exact zeros When l1_ratio = 0.5: equal mix
2
Why ElasticNet beats Lasso with correlated features
When two features X1 and X2 are perfectly correlated, Lasso randomly picks one and drops the other (an arbitrary choice). Ridge part of ElasticNet encourages keeping both by spreading the coefficient between them. This is more honest and more stable.
Example: X1 and X2 are 95% correlated, both important Lasso might give: beta1 = 2.0, beta2 = 0 (arbitrary pick) or: beta1 = 0, beta2 = 2.0 (different run!) ElasticNet gives: beta1 = 1.1, beta2 = 0.9 (stable, shares credit) Ridge gives: beta1 = 1.0, beta2 = 1.0 (stable but no zeros)
3
How to choose alpha and l1_ratio
Never guess these values. Use ElasticNetCV which automatically tries many combinations and picks the best using cross-validation. A good starting strategy: try l1_ratio values in [0.1, 0.5, 0.7, 0.9, 0.95, 1.0] to cover the full range.
from sklearn.linear_model import ElasticNetCV model = ElasticNetCV( l1_ratio = [0.1, 0.5, 0.7, 0.9, 0.95, 1.0], # try many values alphas = None, # auto-generate 100 alpha values cv = 5 # 5-fold cross-validation ) model.fit(X_train, y_train) print(model.alpha_) # best alpha found print(model.l1_ratio_) # best l1_ratio found

!
Common mistakes

✕
Not scaling features. Both L1 and L2 penalties are sensitive to feature scale. Use StandardScaler before ElasticNet — always. This is the single most common mistake.
✕
Setting l1_ratio = 0 and calling it ElasticNet. When l1_ratio = 0, you have pure Ridge. Just use Ridge directly — it has a much faster closed-form solution. ElasticNet with l1_ratio=0 uses slow coordinate descent for no benefit.
✕
Guessing alpha and l1_ratio by hand. Use ElasticNetCV. It tries many combinations automatically with cross-validation. Manual guessing almost never gives the best result.
✕
Using ElasticNet when data is non-linear. Like Ridge and Lasso, ElasticNet is a linear model. If your data has curves, combine it with PolynomialFeatures, or switch to a tree-based model entirely.

★
Key takeaways

Summary
ElasticNet = Ridge penalty + Lasso penalty. The l1_ratio dial controls the mix. 0 = Ridge, 1 = Lasso, 0.5 = equal blend.
ElasticNet inherits sparsity from Lasso (zeros features out) and stability from Ridge (handles correlated features without randomly picking one).
When you're not sure whether to use Ridge or Lasso, just use ElasticNet with ElasticNetCV. It will find the right balance for you.
Always scale features first. StandardScaler before ElasticNet. No exceptions.
Use ElasticNetCV with a list of l1_ratio values like [0.1, 0.5, 0.9, 1.0] to search both hyperparameters at once.
