Right metrics for imbalanced data

1
Never use alone
Accuracy — misleading on imbalanced data
A model predicting ALL majority gets (majority/total) accuracy. For 99:1 data that is 99% accuracy with zero predictive value for the minority class.
accuracy = (TP + TN) / (TP + TN + FP + FN) # 99% accuracy but catches 0% of fraud = useless!
2
Use these
Precision, Recall, and F1
Precision = of predicted positives, how many correct. Recall = of actual positives, how many caught. F1 = harmonic mean. For fraud/disease, recall matters most.
precision = TP / (TP + FP) # of predicted fraud, how many real? recall = TP / (TP + FN) # of actual fraud, how many caught? F1 = 2 * (P * R) / (P + R)
3
Best single metric
ROC-AUC and PR-AUC
ROC-AUC measures discrimination across all thresholds. For extreme imbalance (<5%), use PR-AUC instead. ROC-AUC can look great while PR-AUC is terrible.
ROC-AUC = area under ROC curve (TPR vs FPR) PR-AUC = area under Precision-Recall curve # Use PR-AUC for severe imbalance (<5% minority)
4
Threshold tuning
Move the decision threshold
Default threshold is 0.5. Lower it (e.g. 0.3) to catch more minority cases (higher recall) at the cost of more false alarms (lower precision). Choose based on business cost of each error type.
y_pred = (model.predict_proba(X)[:,1] >= threshold).astype(int) # Find optimal threshold via F1 or cost-sensitive criterion

Common mistakes

x
Optimising for accuracy on imbalanced data. Always use F1, ROC-AUC, or PR-AUC. Report per-class precision and recall.
x
Applying SMOTE before train-test split. SMOTE must only be applied to training data. Applying it first leaks synthetic samples into the test set and inflates metrics.
x
Always defaulting to SMOTE. class_weight='balanced' is simpler, faster, and often equally effective. Try weights first before resampling.
x
Ignoring business cost of errors. False negatives (missed fraud) may cost 1000x more than false positives. Tune threshold based on real cost, not F1.
x
Using ROC-AUC with extreme imbalance. At 0.1% minority, ROC-AUC can be 0.95 while PR-AUC is 0.10. Always report PR-AUC for <5% minority problems.

*
Key takeaways

Summary
Never evaluate imbalanced models on accuracy alone. Use F1, ROC-AUC, and PR-AUC.
Try class_weight='balanced' first — simplest fix, often as good as SMOTE.
SMOTE must only be applied to training data, never before the train-test split.
For severe imbalance (<5%), use PR-AUC instead of ROC-AUC as the primary metric.
Tune the decision threshold based on business cost of each error type.
BalancedRandomForest and EasyEnsemble are often the best out-of-the-box solutions.
