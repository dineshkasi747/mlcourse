Hyperparameter Tuning

Parameters you set before training — not learned from data. Grid Search, Random Search, and Bayesian Optimisation are the three strategies. The right one depends on your compute budget.

20 min Intermediate sklearn · optuna

☷
Grid Search
Exhaustive
Tries every combination of specified values. Guaranteed to find the best in the grid but scales exponentially with parameters.
☢
Random Search
Efficient
Samples randomly from distributions. Finds good solutions faster than grid when only a few parameters matter (which is usually the case).
◆
Bayesian Optimisation
Smart
Builds a surrogate model of the objective function. Uses past results to choose the next point — focuses budget where improvement is likely.

Production tuning workflow

1
Baseline first
Get a baseline with default parameters
Never tune before you have a baseline. Default sklearn parameters are well-chosen starting points. Measure cross-validated score first so you know how much tuning actually helps.
model = RandomForestClassifier() # all defaults baseline = cross_val_score(model, X, y, cv=5).mean() print(f"Baseline: {baseline:.4f}")
2
Coarse search
Random search over a wide range
Start with Random Search over a large parameter space using log-uniform distributions. This is faster than Grid Search and covers the space better. Use n_iter=50-100 with 3-fold CV.
from scipy.stats import loguniform param_dist = { 'n_estimators': [50, 100, 200, 500], 'max_depth': [None, 5, 10, 20], 'min_samples_split': loguniform(0.001, 0.1), } rs = RandomizedSearchCV(model, param_dist, n_iter=50, cv=3)
3
Fine search
Grid search around the best region
Once you know roughly where the optimum is, run a tight Grid Search in that neighbourhood. Use 5-fold CV here since you have fewer combinations.
best = rs.best_params_ # Narrow grid around best param_grid = { 'n_estimators': [best['n_estimators']-50, best['n_estimators'], best['n_estimators']+50], 'max_depth': [best['max_depth']-2, best['max_depth'], best['max_depth']+2], }
4
Final evaluation
Evaluate on held-out test set ONCE
After tuning, fit the best model on all training data and evaluate on the test set exactly once. If you tune using test set feedback you leak information and your estimate is optimistic.
best_model = gs.best_estimator_ best_model.fit(X_train, y_train) # fit on ALL train test_score = best_model.score(X_test, y_test) print(f"Test: {test_score:.4f}") # report THIS number
5
Advanced
Bayesian Optimisation for expensive models
When each model fit takes minutes (XGBoost, neural nets), use Optuna or scikit-optimize. They use past evaluations to intelligently choose the next trial.
import optuna def objective(trial): n = trial.suggest_int('n_estimators', 50, 500) d = trial.suggest_int('max_depth', 2, 20) model = RandomForestClassifier(n_estimators=n, max_depth=d) return cross_val_score(model, X, y, cv=3).mean() study = optuna.create_study(direction='maximize') study.optimize(objective, n_trials=100)

Common mistakes

x
Tuning on the test set. If you use test set feedback to choose hyperparameters, your reported accuracy is optimistic. Use cross-validation on training data only. Evaluate on test set exactly once at the end.
x
Grid searching too many parameters at once. A 5-parameter grid with 5 values each = 5^5 = 3125 fits. With 5-fold CV that is 15,625 model fits. Random Search with n_iter=100 explores the same space in 500 fits.
x
Not using log-scale for LR and regularisation. Learning rate ranges like [0.001, 0.01, 0.1, 1.0] are log-spaced. Using linear spacing [0.1, 0.2, 0.3] misses the most important region near 0.
x
Tuning before feature engineering. Hyperparameter tuning gives marginal gains. Better features give large gains. Always do EDA and feature engineering first.
x
Overfitting the validation set. Running hundreds of trials will eventually find params that fit the validation noise. Use nested CV or a separate validation set for final reporting.

*
Key takeaways

Summary
Grid Search is exhaustive but exponential. Use it only for 1-2 parameters with few values.
Random Search is usually better than Grid Search. Sample from distributions, not fixed grids. Use log-uniform for LR and regularisation.
Bayesian Optimisation (Optuna, scikit-optimize) is best when evaluations are expensive. It learns from past trials to choose the next point.
Never use test set for tuning. Evaluate on test set exactly once after all tuning is done.
Always start with a baseline using default params. Tuning rarely improves more than 1-3% on well-prepared data.
