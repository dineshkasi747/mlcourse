DBSCAN

DBSCAN finds clusters based on density, not distance to a centre. It can find clusters of any shape, automatically detects outliers as noise, and you don't need to pick K upfront — the algorithm finds K itself.

⌛ 20 min ◆ Intermediate sklearn

🌌
Intuition — finding dense neighbourhoods

Imagine you are looking at stars in the night sky. Some stars form tight constellations (clusters). Some stars are isolated (noise / outliers). You wouldn't describe a constellation by its centre — you'd say "these stars are close together and connected." That is exactly what DBSCAN does.

DBSCAN asks one simple question for every point: "Are there at least min_samples other points within distance eps of me?" If yes, that point is a core point and starts or joins a cluster. If no but it's near a core point, it's a border point. If completely isolated, it's noise (label = -1).

💡

Three types of points: Core points — have enough neighbours within eps. They are the heart of clusters.
Border points — within eps of a core point but don't have enough neighbours themselves. On the edge of a cluster.
Noise points — not within eps of any core point. These are outliers. DBSCAN labels them -1.

Hyperparameter tuning — eps and min_samples

DBSCAN has two parameters. eps controls the radius of each point's neighbourhood. min_samples controls how many neighbours are needed to be a core point. Getting these right is the main challenge.

DBSCAN vs K-Means — when to use which

DBSCAN wins when...
✓ Clusters have irregular shapes (moons, rings)
✓ You don't know K upfront
✓ Data has outliers you want to detect
✓ Clusters have very different sizes
✓ Geographic/spatial clustering
K-Means wins when...
✓ Clusters are round and compact
✓ You know K upfront
✓ Dataset is large (K-Means is faster)
✓ All points should belong to a cluster
✓ High-dimensional data
Hierarchical wins when...
✓ You want to explore all cluster levels
✓ Dataset is small (<10k points)
✓ You need a dendrogram for interpretation
✓ Uncertain about K
✓ Need reproducible results

How DBSCAN works — step by step

1
Define the neighbourhood of each point
For every point p, the eps-neighbourhood N(p) is all points within distance eps. This is just a circle of radius eps around p.
N_eps(p) = { q in X | distance(p, q) <= eps } A point p is a CORE POINT if: |N_eps(p)| >= min_samples (it has enough neighbours inside its circle)
2
Classify every point
Every point gets one of three labels based on its neighbourhood. This classification determines whether it starts/joins a cluster or becomes noise.
Core point: |N_eps(p)| >= min_samples -> heart of a cluster Border point: p is in N_eps(q) for some core q, but |N_eps(p)| < min_samples Noise point: not core, not border -> label = -1 (outlier)
3
Expand clusters from core points
Pick any unvisited core point. Assign it a new cluster label. Add all its neighbours to the cluster. For each neighbour that is also a core point, add their neighbours too. Keep expanding until no more core points can be reached. Then move to the next unvisited point.
Algorithm: for each unvisited point p: mark p as visited if p is NOT a core point: label p as noise (might change later if neighbour is core) else: create new cluster C add p to C queue = N_eps(p) while queue is not empty: q = queue.pop() if q is unvisited: mark q visited if q not in any cluster: add q to C if q is a core point: queue += N_eps(q)

Common mistakes

✕
Not scaling features. DBSCAN uses Euclidean distance. A feature in thousands dominates. Always StandardScaler first. After scaling, eps values around 0.3–1.5 usually work well.
✕
Guessing eps. Use the k-distance graph: compute each point's distance to its k-th nearest neighbour (k=min_samples), sort them, and plot. The "elbow" in that plot is a good eps value.
✕
Using DBSCAN on high-dimensional data. In high dimensions, all points become roughly equidistant (the curse of dimensionality). eps becomes meaningless. Reduce dimensions first with PCA, then apply DBSCAN.
✕
Treating noise points as a cluster. DBSCAN labels outliers as -1. This does NOT mean they form a cluster. They are individual anomalies. Never group noise points together.
✕
Clusters with very different densities. DBSCAN uses one global eps. If cluster A is very dense and cluster B is sparse, no single eps works for both. Use HDBSCAN (a variant) which handles varying densities automatically.

★
Key takeaways

Summary
DBSCAN finds clusters based on density. It needs two parameters: eps (neighbourhood radius) and min_samples (minimum neighbours to be a core point).
Three point types: core points (dense enough), border points (near a core), and noise points (outliers, label = -1).
DBSCAN can find clusters of any shape and automatically detects outliers. Unlike K-Means, you don't need to specify K.
Always scale features first. Use the k-distance elbow plot to choose eps. Start with min_samples = 2 × n_features as a rule of thumb.
For varying-density clusters, use HDBSCAN — a hierarchical extension that handles different densities automatically.
