Voting Ensembles

Ask several different models to make a prediction. Then combine their answers. Even if each model is only okay, the group together is usually better — because their errors happen in different places and cancel each other out.

⌛ 15 min ◆ Beginner sklearn

👥
Intuition — ask a crowd, not just one expert

Imagine you need to decide if a patient has a disease. You ask five doctors. Three say yes, two say no. You go with the majority — yes. Even if each doctor is only 70% accurate, the group together is much more accurate, because they make different mistakes.

That is exactly what a Voting Ensemble does. You train several completely different models (a decision tree, a logistic regression, a KNN...). Each model votes on the answer. The majority wins.

💡

Why does it work? If Model A is wrong on examples 1, 3, 7 and Model B is wrong on examples 2, 5, 9, their errors don't overlap. When you combine them, most examples get at least 2 correct votes and only 1 wrong vote. The majority vote is correct. This is called error diversity — the key ingredient of all ensemble methods.

Hard voting vs Soft voting

There are two ways to combine predictions:

  • Hard voting — each model predicts a class label (0 or 1). You count the votes and pick the majority. Simple, works with any model.
  • Soft voting — each model outputs a probability (e.g. 0.8 for Class A). You average the probabilities and pick the highest. Usually better because it uses more information.

Weighted voting — give better models more say

Why should a model with 60% accuracy count the same as one with 90% accuracy? With weighted voting, you assign each model a weight. Better models get higher weights and influence the final decision more.

The math — simple majority rule

1
Hard voting — majority class wins
Each model predicts a class. Count how many models predicted each class. The class with the most votes wins. Ties are broken by the first class.
Models: DT=Class A, LR=Class A, KNN=Class B, SVM=Class A, NB=Class B Hard vote count: Class A = 3, Class B = 2 Final prediction: Class A (majority wins)
2
Soft voting — average probabilities
Each model outputs a probability for each class. Average them across models. The class with the highest average probability wins. This uses more information than hard voting.
Model probabilities for Class A: DT = 0.80, LR = 0.72, KNN = 0.35, SVM = 0.91, NB = 0.44 Average probability: (0.80+0.72+0.35+0.91+0.44) / 5 = 0.644 Final prediction: Class A (average > 0.5)
3
Weighted soft voting
Give better models higher weights. The final probability is a weighted average. The weights don't need to sum to 1 — sklearn normalises them automatically.
Weights: DT=1, LR=2, KNN=1, SVM=3, NB=1 (total = 8) Weighted average: = (1*0.80 + 2*0.72 + 1*0.35 + 3*0.91 + 1*0.44) / 8 = (0.80 + 1.44 + 0.35 + 2.73 + 0.44) / 8 = 5.76 / 8 = 0.72 Final prediction: Class A (SVM's high confidence is amplified)

Common mistakes

✕
Using too many similar models. If all 5 models are decision trees with the same settings, they will make the same mistakes. You need diversity. Mix different model types: trees, linear models, KNN, etc.
✕
Using hard voting when models output probabilities. Soft voting almost always performs better because it uses more information. Only use hard voting when some models don't have a predict_proba method.
✕
Not tuning individual models first. A voting ensemble of bad models is a bad ensemble. Tune each base model separately with cross-validation before combining them.
✕
Expecting big gains on easy problems. If one model already gets 98% accuracy, voting won't help much. Ensembles shine most on hard problems where individual models are around 70-85% accuracy.

★
Key takeaways

Summary
A voting ensemble combines predictions from multiple models. The key requirement is diversity — models should make different errors.
Soft voting (average probabilities) is almost always better than hard voting (count class labels). Use soft voting whenever your models support predict_proba.
Weighted voting lets better models have more influence. Set weights proportional to each model's cross-validation accuracy.
Voting ensembles are simple and fast to implement. For better results with learned weights, use Stacking (next topic).
sklearn's VotingClassifier and VotingRegressor make this one-liner easy. Just pass a list of (name, model) tuples.
