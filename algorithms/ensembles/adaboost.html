The AdaBoost algorithm

AdaBoost runs T rounds. Each round trains a stump on the current sample weights, computes its confidence (alpha), then re-weights samples so the next stump focuses on what the current one got wrong.

1
Initialisation
Uniform sample weights
Every sample starts with equal weight 1/n. Weights always sum to 1 throughout training. Think of them as a probability distribution over the training set.
w_i = 1/n for i = 1 .. n
2
Weak learner training
Train a weighted decision stump
Fit a stump that minimises weighted misclassification error. Points with high weight dominate the split criterion — the stump must get them right.
h_t = argmin_h sum_i w_i * 1[ h(x_i) != y_i ] epsilon_t = sum_i w_i * 1[ h_t(x_i) != y_i ] # weighted error rate
3
Stump confidence
Compute alpha (voting weight)
Alpha measures how much this stump should count in the final vote. Better stumps (low error) get high alpha. A 50%-error stump gets alpha=0 and is ignored. A >50% stump gets negative alpha — it votes backwards!
alpha_t = (lr / 2) * ln( (1 - epsilon_t) / epsilon_t ) # Examples: # epsilon = 0.05 -> alpha = +1.47 (very trusted) # epsilon = 0.30 -> alpha = +0.42 (moderately trusted) # epsilon = 0.50 -> alpha = 0.00 (ignored — no better than random) # epsilon = 0.70 -> alpha = -0.42 (reversed — votes opposite)
4
Weight update
Boost misclassified, shrink correct
Misclassified samples get weight * e^alpha (>1, grows). Correct samples get weight * e^-alpha (<1, shrinks). Then renormalise so weights still sum to 1. Next stump inherits a harder problem.
w_i := w_i * exp( -alpha_t * y_i * h_t(x_i) ) # y_i, h_t in {-1, +1} # misclassified: y_i * h_t = -1 => multiply by e^(+alpha) (BIGGER) # correct: y_i * h_t = +1 => multiply by e^(-alpha) (smaller) w_i := w_i / sum(w) # renormalise to sum = 1
5
Final prediction
Weighted majority vote
Each stump casts a vote weighted by its alpha. Take the sign of the sum. High-alpha stumps dominate; low-alpha stumps barely matter. The ensemble is much stronger than any individual stump.
H(x) = sign( sum_t alpha_t * h_t(x) ) # Repeat steps 2-4 for T rounds, then use this formula for all predictions.

Common mistakes

x
Using AdaBoost with noisy/outlier data. Outliers are always misclassified, so they accumulate huge weights and dominate training. AdaBoost is far more sensitive to outliers than GBM. Use GBM or XGBoost with regularisation for noisy data.
x
Not scaling features when using non-tree base estimators. AdaBoost with decision stumps doesn't need scaling. But if you change the base estimator to SVM or Logistic Regression, scale first.
x
Using very deep trees as weak learners. AdaBoost works best with shallow trees (depth 1-3). Deep trees are strong learners and defeat the purpose of boosting. Use GBM if you need depth.
x
Expecting AdaBoost to match GBM/XGBoost. AdaBoost is simpler and often slightly weaker on real datasets. Use it to understand boosting, then switch to GBM/XGBoost for production.

*
Key takeaways

Summary
AdaBoost re-weights samples each round. Misclassified points get higher weight, forcing the next stump to focus on them.
Alpha measures stump quality. High for accurate stumps, zero for random, negative for worse-than-random. Final prediction = sign of alpha-weighted sum.
AdaBoost is sensitive to outliers. Noisy points accumulate huge weights. GBM is more robust in practice.
Use decision stumps (max_depth=1) as the base learner. Stumps are the classic and most interpretable choice.
For production accuracy, prefer GBM, XGBoost, or LightGBM over AdaBoost.
