Information Gain

The reduction in entropy after a split. The tree greedily picks the split with the highest IG at every node — the question that teaches you the most.

What is Information Gain?

Information Gain measures how much a split reduces entropy. The tree picks the split with the highest IG at each node — the question that teaches you the most about class labels.

IG = H(parent) − [nL/n · H(left) + nR/n · H(right)]

💡

Mental model: IG answers "After this split, how much less uncertain am I about the class?" A perfect split that separates all classes gives IG = full parent entropy.

Best IG per feature — Iris dataset

Formula

1
Parent entropy
H(parent) = -Σ pᵢ log₂(pᵢ)
2
Weighted child entropy
H_child = (nL/n)·H(left) + (nR/n)·H(right)
3
Information Gain
IG(j,t) = H(parent) - H_child [always >= 0]
4
Best split selection
(j*,t*) = argmax_{j,t} IG(j,t)

★
Key takeaways

Summary
IG = entropy reduction from parent to children. Higher = better split. Always ≥ 0.
The tree tries every feature × every threshold and picks the maximum IG. O(n·d) per node.
IG is biased toward high-cardinality features. Gini is slightly less biased in practice.
The root split feature is the single most informative feature in the entire dataset.
