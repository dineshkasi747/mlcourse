Entropy

Entropy measures how much uncertainty exists in a set of labels. A pure node (all one class) has entropy = 0. A perfectly mixed node has maximum entropy. Decision trees split data to reduce entropy as fast as possible.

15 min Beginner

What is entropy? — a coin flip intuition

Entropy answers: "How surprised will I be by the next outcome?" If I flip a fair coin, I'm maximally uncertain — entropy is high. If the coin always lands heads, I'm never surprised — entropy is zero.

In decision trees, each node holds a bag of training samples with labels. High entropy = labels are mixed. Low entropy = labels are mostly one class — confident prediction.

The formula — where does -p·log2(p) come from?

The entropy formula comes from information theory. Each term -pi·log2(pi) is the expected surprise of class i.

H(S) = -Sum_i p_i * log2(p_i)
  • p_i = proportion of class i in node S
  • log2(p_i) = how many bits to encode this class (negative for p less than 1)
  • -p_i * log2(p_i) = expected surprise for class i (0 when p=0 or p=1)
  • Sum over all classes = total average surprise in the node
ok

Why log2? Because we measure information in bits. 1 bit = 1 yes/no question. A fair coin flip gives exactly 1 bit of information. A 4-sided die gives log2(4) = 2 bits. Using log2 makes the numbers intuitive.

Key takeaways

Summary
Entropy measures impurity and uncertainty in a node. 0 = pure, log2(K) = maximum disorder for K classes.
Formula: H = -Sum pi*log2(pi). Each term is the expected information content of that class.
Decision trees use entropy to choose splits — pick the feature+threshold that maximises Information Gain (reduction in entropy after the split).
Binary entropy max = 1 bit. Multi-class entropy max = log2(K) bits.
Entropy and Gini impurity give very similar results. Entropy is ~2x slower to compute but may give slightly more balanced splits.
