Gini Impurity

Gini impurity measures how mixed a group of items is. A group of all one class has Gini = 0 (perfectly pure). A perfectly mixed group has Gini = 0.5. Decision trees pick the split that reduces Gini the most.

⌛ 15 min ◆ Beginner Decision Trees

●
Intuition — how mixed is this bag?

Imagine a bag of coloured balls. If all balls are the same colour, the bag is pure — Gini = 0. If the bag has half red and half blue, it is maximally impure — Gini = 0.5. Gini impurity is just a number that measures how mixed the bag is.

Decision trees use Gini to decide where to split data. At each node, the tree tries every possible split and picks the one that results in the lowest Gini impurity in the child nodes.

Simple rule: Gini = 0 means the node is perfect (all one class, decision tree is done). Gini = 0.5 means the worst case for 2 classes — totally random. The decision tree always tries to reduce Gini towards 0 with each split.

Gini vs Entropy — which one to use?

Decision trees can use either Gini impurity or entropy (information gain) to pick splits. They usually give the same results — the choice rarely matters in practice. Gini is slightly faster to compute (no logarithm). Entropy can sometimes prefer more balanced splits. sklearn uses Gini by default.

Gini vs Entropy curves side by side
Both curves peak at 50/50 and reach 0 at pure nodes. Entropy (scaled to 0-0.5) is slightly higher in the middle — it "punishes" impurity a bit more. In practice, both give nearly identical trees.

With K classes, the maximum Gini = 1 - 1/K. With 2 classes max = 0.5. With 4 classes max = 0.75.

The formula — simple and fast

1
The Gini formula
For a node with K classes, compute the proportion of each class (p_i). Gini = 1 minus the sum of squared proportions. That's it.
Gini(node) = 1 - sum(p_i^2) for each class i Example: node has 4 blues, 1 red (5 total) p_blue = 4/5 = 0.8 p_red = 1/5 = 0.2 Gini = 1 - (0.8^2 + 0.2^2) = 1 - (0.64 + 0.04) = 1 - 0.68 = 0.32
2
Weighted Gini after a split
After splitting a node into left and right children, compute the weighted average of child Gini values. The weights are the fraction of samples that went to each child.
Weighted Gini = (n_left/n) * Gini(left) + (n_right/n) * Gini(right) Example: 10 samples split into left=7, right=3 Left node: 5 blue, 2 red -> Gini = 1-(5/7)^2-(2/7)^2 = 0.408 Right node: 2 blue, 1 red -> Gini = 1-(2/3)^2-(1/3)^2 = 0.444 Weighted = (7/10)*0.408 + (3/10)*0.444 = 0.286 + 0.133 = 0.419
3
Gini Gain (Information Gain)
The improvement from a split is the parent Gini minus the weighted child Gini. The decision tree picks the split with the highest Gini Gain.
Gini Gain = Gini(parent) - Weighted Gini(children) Continuing example: Parent Gini = 0.490 (7 blue, 3 red in parent) Weighted Gini after split = 0.419 Gini Gain = 0.490 - 0.419 = 0.071 The tree tries ALL possible splits and picks the one with the HIGHEST Gini Gain.
4
Why Gini not accuracy?
You might ask: why not just measure classification accuracy? Because accuracy is not differentiable and does not account for how confident the prediction is. A node predicting 51% class A has the same accuracy as 99% class A, but Gini correctly sees the second as far purer.
Node A: 51% class A, 49% class B -> Accuracy = 51% Gini = 0.4998 Node B: 99% class A, 1% class B -> Accuracy = 99% Gini = 0.0198 Gini correctly shows B is much purer. Accuracy would give both the same label ("Class A wins") but miss the huge difference in confidence.

Key takeaways

Summary
Gini impurity = 1 - sum(p_i²). It measures how mixed a node is. 0 = perfectly pure, 0.5 = maximally mixed (2 classes).
Decision trees pick the split that gives the lowest weighted Gini across children — equivalent to maximising Gini Gain.
Gini and Entropy almost always choose the same splits. Gini is faster (no log). sklearn uses Gini by default.
With K classes, max Gini = 1 - 1/K. More classes = higher possible impurity ceiling.
A leaf node with Gini = 0 is a perfect prediction — all training samples in that leaf belong to one class.
