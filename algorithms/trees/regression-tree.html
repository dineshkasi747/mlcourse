Decision Trees · Regression

Regression Tree

Decision trees are not just for classification. When your target is a continuous number (house price, temperature, salary), a regression tree splits the feature space into rectangles and predicts the mean of training points in each region. Same tree structure — different split criterion and different leaf output.

⏱ 16 min ◆ Beginner

⇄
Classification tree vs Regression tree — two key differences

Classification Tree
  • Target: a class label (spam / not spam, cat / dog)
  • Split criterion: Gini impurity or Entropy — measures class mixing
  • Leaf output: the majority class in that region
  • Evaluation metric: Accuracy, F1, ROC-AUC
Regression Tree
  • Target: a continuous number (price, weight, score)
  • Split criterion: MSE (Mean Squared Error) — measures spread of y values
  • Leaf output: the mean of y values in that region
  • Evaluation metric: MAE, RMSE, R²
💡

The split logic is identical. At each node, the tree tries every feature and every threshold. It picks the split that most reduces MSE (instead of Gini). The only difference is what we measure and what we output at the leaves.

The key limitation — step-function predictions

Regression trees produce step-function predictions. Every point in a leaf gets the same predicted value (the leaf mean). This means regression trees can never extrapolate beyond the range of training data — and struggle with smooth continuous relationships.

For smoother predictions, use gradient boosting or random forest regression — both build many regression trees and combine them. Single regression trees are mainly used as interpretable baselines or as components inside ensembles.

Key takeaways

Summary
Regression trees split to minimise MSE (not Gini). Leaf output is the mean of y values in that region.
Predictions are step functions — every point in a leaf gets the same value. Cannot extrapolate.
Same hyperparameters as classification trees: max_depth, min_samples_leaf, min_samples_split control complexity.
Single regression trees are mainly used as interpretable baselines or inside ensemble methods (Random Forest, Gradient Boosting).
