Naive Bayes

Apply Bayes' theorem with one naive assumption: every feature is independent given the class. Absurdly simple. Surprisingly effective. The backbone of spam filters for two decades.

⏱ 18 min ◆ Beginner numpy · sklearn

?
The core idea — what does "naive" mean?

Naive Bayes answers: "Given these features, what's the most probable class?" It uses Bayes' theorem to flip the question: instead of asking P(class | features), it computes P(features | class) × P(class) — which is much easier to estimate from training data.

The "naive" part: it assumes every feature is conditionally independent given the class. In spam detection, this means "free" and "money" each contribute independently to the spam score — ignoring that they often appear together. This assumption is almost always wrong in reality, yet the classifier still works remarkably well.

Real-world intuition: A doctor sees symptoms (fever, cough, fatigue) and computes: "Given these symptoms, what disease is most probable?" They implicitly multiply prior disease rates by how likely each symptom is per disease — exactly what Naive Bayes does.

The "naive" assumption — visualised

The independence assumption says: knowing "free" appears tells us nothing extra about whether "money" appears, once we know the class. This is almost always false — but the classifier survives anyway because it only needs to get the ranking of posteriors right, not their exact values.

✗ Reality: words co-occur
"free" and "money" appear together 60% of the time in spam. They are correlated.
✓ Naive Bayes assumption
Naive Bayes treats them as independent: P("free" AND "money") = P("free") × P("money").
⚠

Why does it still work? Even when features are correlated, the model usually gets the right class ranking. The absolute posterior probabilities are wrong (they're too extreme), but the argmax is usually correct. When you need calibrated probabilities, use logistic regression instead.

Three variants — which to use when

The choice of variant depends entirely on what type of features you have.

Gaussian NB
Continuous
Assumes features follow a Gaussian distribution within each class. Fits mean and variance per feature per class.
Use for: real-valued features like height, weight, temperature, sensor readings.
Multinomial NB
Count data
Features are word counts or frequencies. Uses a multinomial distribution — probability of each word given the class.
Use for: text classification, document classification, TF-IDF features.
Bernoulli NB
Binary
Each feature is binary (0 or 1). Considers both presence AND absence of features — explicitly penalises missing words.
Use for: short texts, document presence/absence, binary feature vectors.

Laplace smoothing — the zero-probability problem

If a word never appeared in spam during training, its likelihood P("word"|spam) = 0. Multiplying by zero kills the entire posterior — one unseen word makes the class probability zero no matter how strong the other evidence is. Laplace smoothing adds α (usually 1) to every count, preventing zeros.

Full derivation — step by step

1
Start with Bayes' theorem
The posterior probability of class C given features x is proportional to prior times likelihood.
P(C | x₁,x₂,...,xₙ) = P(x₁,x₂,...,xₙ | C) · P(C) / P(x₁,x₂,...,xₙ)
2
Apply the naive independence assumption
P(x₁,x₂,...,xₙ | C) = P(x₁|C) · P(x₂|C) · ... · P(xₙ|C). The joint becomes a product of marginals.
P(C | x) ∝ P(C) · ∏ᵢ P(xᵢ | C)
3
Denominator is constant — drop it
P(x₁,...,xₙ) is the same for all classes. We only need to compare numerators, so we drop it.
ŷ = argmax_C P(C) · ∏ᵢ P(xᵢ | C)
4
Take log to avoid underflow
Products of many small probabilities underflow to 0 in floating point. Log turns products into sums — mathematically equivalent, numerically stable.
ŷ = argmax_C [ log P(C) + Σᵢ log P(xᵢ | C) ]
5
Estimate parameters from training data
Prior = class frequency. For Multinomial NB, likelihood = word count + α divided by class total + V·α (Laplace smoothing).
P(C) = count(C) / N P(word | C) = (count(word,C) + α) / (count(C) + α·|V|)

Common mistakes

✕
Using alpha=0 on text data. Without smoothing, any word not seen in training silences the entire class probability. Always use alpha >= 0.1 for text.
✕
Using Gaussian NB on count/text data. Gaussian NB assumes normally distributed features. Word counts are not Gaussian — use Multinomial NB for counts, Bernoulli NB for presence/absence.
✕
Trusting the output probabilities too much. Because of the independence assumption, NB posteriors are often overconfident (close to 0 or 1). They rank correctly but aren't well-calibrated. Use CalibratedClassifierCV if you need reliable probability estimates.
✕
Forgetting to normalise features for Gaussian NB. Unlike tree-based models, Gaussian NB fits per-class Gaussians — correlated and highly skewed features will mislead it. Log-transform skewed numeric features first.
✕
Dismissing Naive Bayes as "too simple". For text classification, NB is often competitive with or better than SVM and neural networks on small datasets. It trains in milliseconds and is fully interpretable. Always include it as a baseline.

★
Key takeaways

Summary
Naive Bayes applies Bayes' theorem with the independence assumption: features are conditionally independent given the class. Wrong assumption, surprisingly effective model.
Classification = argmax over [log P(C) + sum of log P(xᵢ|C)]. This is just counting and adding — extremely fast training and prediction.
Three variants: Gaussian (continuous), Multinomial (counts/text), Bernoulli (binary). Match the variant to your feature type.
Laplace smoothing (alpha > 0) is essential for text data. It prevents unseen words from zeroing out class probabilities.
Probabilities are not calibrated — rankings are correct but values are overconfident. For real probabilities, use logistic regression or calibrate NB output.
Use NB as a fast, interpretable baseline. It often achieves 90%+ of the best model's accuracy in a fraction of the time.
