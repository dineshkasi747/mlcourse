Classification Metrics

Accuracy alone lies. A model that predicts "not sick" for every patient gets 99% accuracy on a rare disease dataset — but catches zero sick people. You need Precision, Recall, F1, ROC-AUC to understand what your model actually does.

Each metric explained — with the formula

Formula visualiser — what each metric measures
💡

Precision = "Of all the positives I predicted, how many were actually positive?" (TP / (TP + FP)) — minimise false alarms.
Recall = "Of all the actual positives, how many did I catch?" (TP / (TP + FN)) — minimise missed positives.
F1 = harmonic mean of precision and recall — balances both. Use when you can't sacrifice either.

θ
Threshold — the dial that trades precision for recall

Your model outputs a probability (0 to 1). The threshold decides what probability counts as "positive". Lower the threshold → catch more positives (higher recall) but also more false alarms (lower precision). Raise it → fewer false alarms but miss more positives.

This is a business decision, not a model parameter. Drag the threshold below and watch precision, recall, and F1 shift in real time on a simulated dataset.

ROC
ROC curve and AUC — threshold-independent evaluation

The ROC (Receiver Operating Characteristic) curve shows all possible precision-recall tradeoffs at once by sweeping the threshold from 0 to 1 and plotting True Positive Rate vs False Positive Rate. The area under this curve (AUC) summarises the model's discriminative power in a single number.

AUC = probability that the model ranks a random positive higher than a random negative. AUC = 0.5 means random guessing. AUC = 1.0 means perfect separation.

Precision-Recall curve — better for imbalanced data

When classes are very imbalanced (e.g., 1% fraud, 99% normal), ROC-AUC looks optimistic because the large true negative pool inflates performance. The PR curve focuses only on the positive class — it is the better choice for imbalanced datasets.

Rule of thumb: Use ROC-AUC when classes are roughly balanced. Use PR-AUC (Average Precision) when the positive class is rare (fraud, disease, rare events). A perfect PR-AUC = 1.0, random baseline = positive_rate% (e.g. 10% for a 10% positive rate).

Common mistakes

✕
Using accuracy on imbalanced data. If 99% of samples are negative, a model that always says "negative" gets 99% accuracy. Use F1, PR-AUC, or ROC-AUC instead.
✕
Not picking a threshold based on business context. 0.5 is not always the right threshold. Cancer screening needs low threshold (catch everything). Loan approval needs high threshold (be very sure).
✕
Optimising precision OR recall, not both. Predicting all as positive gives 100% recall. Predicting only the single most-confident case as positive gives 100% precision. Always report both, or use F1.
✕
Using ROC-AUC on very imbalanced data. ROC-AUC can look great (0.97) while the model has poor precision on rare positives. Use PR-AUC / Average Precision when positives are rare.

★
Key takeaways

Summary
All metrics come from 4 numbers: TP, TN, FP, FN. Build the confusion matrix first.
Accuracy is misleading on imbalanced data. Always check Precision, Recall, F1 too.
Threshold is a business decision — low for safety-critical (catch everything), high when false alarms are costly.
ROC-AUC = best for balanced datasets. PR-AUC = best for rare positive class (fraud, disease).
F1 score = harmonic mean of precision and recall. Use when you need one number that balances both.
