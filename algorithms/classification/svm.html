Support Vector Machine

Find the widest possible street between two classes. The data points sitting on the kerb are your support vectors — remove any other point and the boundary doesn't change.

⏱ 22 min ◆ Intermediate numpy · sklearn

💡
The core idea — maximum margin

Imagine you're drawing a line to separate red and blue dots on paper. There are infinitely many valid lines — so which one do you pick?

SVM picks the one that leaves the most breathing room on both sides. It finds the hyperplane that maximises the margin — the gap between the two classes. A wider margin means the model generalises better to unseen data.

H
Hyperplane
The decision boundary. A line in 2D, a plane in 3D, a hyperplane in N-D. Points on one side = Class A, other side = Class B.
★
Support Vectors
The closest training points to the hyperplane. Only these points define the boundary. Delete any other point — boundary stays identical.
←→
Margin
The total width of the gap between the two margin lines. SVM maximises this. Wider margin = better generalisation.
💡

Mental model: Think of SVM as finding the fattest possible road between two cities. The support vectors are the houses right on the road's edge. You can demolish every other house and the road stays exactly the same.

The kernel trick — solving non-linear problems

What if no straight line can separate the classes? The kernel trick maps data into a higher dimension where it becomes linearly separable — without ever computing the transformation explicitly.

The key insight: SVM only ever needs dot products between points — never the coordinates themselves. So we can replace every dot product with a kernel function K(x, y) that secretly computes the dot product in high-dimensional space.

Linear
K(x,y) = xᵀy
No transformation. Works when data is already separable.
RBF / Gaussian
exp(-γ‖x-y‖²)
Most powerful. Maps to infinite dimensions. Default choice.
Polynomial
(γxᵀy + r)ᵈ
Degree-d polynomial features. Good for image data.
Sigmoid
tanh(γxᵀy + r)
Neural-network-like. Rarely outperforms RBF in practice.

Math — what SVM actually optimises

SVM solves a constrained optimisation problem. Here's the full derivation broken into digestible steps.

1
The hyperplane equation
A hyperplane is defined by a weight vector w and bias b. Points where the equation equals 0 lie on the boundary.
f(x) = wᵀx + b = 0
2
Margin width
The perpendicular distance from a point to the hyperplane is |wᵀx + b| / ‖w‖. The total margin (gap between the two support hyperplanes) is:
margin = 2 / ‖w‖
3
Maximise margin = minimise ‖w‖
Maximising 2/‖w‖ is equivalent to minimising ‖w‖, which is equivalent to minimising ½‖w‖² (easier calculus). Subject to: every point correctly classified with margin ≥ 1.
min ½ ‖w‖² subject to: yᵢ(wᵀxᵢ + b) ≥ 1 ∀i
4
Soft margin — allow violations with cost C
Real data is messy. Introduce slack variables ξᵢ ≥ 0 to allow points to be on the wrong side, penalised by C.
min ½ ‖w‖² + C · Σ ξᵢ subject to: yᵢ(wᵀxᵢ + b) ≥ 1 - ξᵢ, ξᵢ ≥ 0
5
Dual form — where kernels enter
Solving the Lagrangian dual gives a form that only uses dot products xᵢᵀxⱼ — which we replace with kernel K(xᵢ, xⱼ). Most αᵢ = 0; only support vectors have αᵢ > 0.
max Σ αᵢ - ½ Σᵢ Σⱼ αᵢ αⱼ yᵢ yⱼ K(xᵢ, xⱼ) subject to: 0 ≤ αᵢ ≤ C, Σ αᵢ yᵢ = 0
✓

Why the dual matters: The primal has D+1 variables (one per feature). The dual has N variables (one per training point). When D is huge (e.g., infinite-dimensional with RBF), the dual is still tractable — this is what makes the kernel trick work.

Common mistakes

✕
Not scaling features. SVM computes distances — if salary (0-100k) and age (0-100) are unscaled, salary dominates. Always StandardScaler before SVM. This is the #1 beginner mistake.
✕
Using default C and γ without tuning. The default sklearn values work for some datasets but not all. Always GridSearchCV or RandomizedSearchCV. The difference can be 60% → 95% accuracy.
✕
Applying SVM to 100k+ samples. SVM training is O(N² to N³). At 100k samples it can take hours. Use LinearSVC (O(N)), SGDClassifier, or switch to Random Forest/XGBoost.
✕
Treating SVM output as probability. svm.predict_proba() works but uses Platt scaling — an expensive calibration step. If you need calibrated probabilities, Logistic Regression is faster and more natural.
✕
Ignoring class_weight for imbalanced data. SVM is sensitive to class imbalance. Always set class_weight='balanced' or manually set weights when classes are skewed.

★
Key takeaways

Summary
SVM finds the hyperplane that maximises the margin between classes. Only support vectors matter — all other points are irrelevant to the boundary.
C controls the margin/error tradeoff. Small C = wide margin (more misclassifications allowed). Large C = hard margin (few errors, risk of overfitting).
The kernel trick maps data to higher dimensions where linear separation is possible — without computing the full transformation. RBF is the default choice.
γ (gamma) controls how far each training point's influence reaches. High γ = spiky, overfitted boundary. Low γ = smooth, generalised boundary.
Always scale features before SVM. Feature scaling is critical — not optional.
SVM excels on high-dimensional data (text, images, genomics) but is slow on N > 50k. Use LinearSVC for large datasets.
